EvaluationEntitlementsSigmaReportingFinancial ConnectionsTaxIdentityCryptoClimateForwardingPrivacyWebhooks# [Create a CVC update token](/api/tokens/create_cvc_update)Ask about this sectionCopy for LLMView as MarkdownCreates a single-use token that represents an updated CVC value that you can use for [CVC re-collection](/payments/accept-a-payment-synchronously#web-recollect-cvc). Use this token when [you confirm a card payment](/api/payment_intents/confirm#confirm_payment_intent-payment_method_options-card-cvc_token) or use a saved card on a PaymentIntent with confirmation_method: manual.
For most cases, use our [JavaScript library](/js/tokens/create_token?type=cvc_update) instead of using the API. For a PaymentIntent with confirmation_method: automatic, use our recommended [payments integration](/payments/save-during-payment#web-recollect-cvc) without tokenizing the CVC value.
### Parameters#### cvc_updateobjectRequiredThe updated CVC value this token represents.
Show child parameters### ReturnsReturns the created CVC update token if it’s successful. Otherwise, this call raises [an error](#errors).
POST /v1/tokens```
`curl https://api.stripe.com/v1/tokens \  -u "sk_test_BQokikJ...2HlWgH4olfQ2sk_test_BQokikJOvBiI2HlWgH4olfQ2:" \  -d "cvc_update[cvc]=123"`
```Response```

Use this when you create or update persons associated with a Connect account. Learn more about [account tokens](/connect/account-tokens).
You can only create person tokens with your application’s publishable key and in live mode. You can use your application’s secret key to create person tokens only in test mode.
### Parameters- #### personobjectRequiredInformation for the person this token represents.
