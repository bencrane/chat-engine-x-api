# BrandName GraphQL Object Type

## Overview

The `BrandName` object represents "the customer-facing version of the name that best represents the business" within Enigma's GraphQL API.

## Data Sources

The brand name information is compiled from:
- Publicly available business data and listings
- Privately verified business information

## Methodology

Enigma determines the optimal business name through a multi-step process:

- High-quality data sources are ranked by dataset quality and frequency to identify the most commonly used business name
- For person-led businesses, agent or provider names may serve as the brand name, with affiliated relationships to parent companies
- Franchise operations and multi-unit businesses maintain separate brand names with documented affiliate relationships

## Fields

| Field | Type | Description |
|-------|------|-------------|
| `name` | `String` | The customer-facing business name, generated by combining and ranking names from multiple data sources |
| `nameFullTextSearchVector` | `String` | Search vector field for full-text name queries |
| `id` | `UUID!` | Unique identifier (non-null) |
| `firstObservedDate` | `String` | Date when the name was first detected |
| `lastObservedDate` | `String` | Date when the name was most recently confirmed |
| `internalId` | `String` | Internal reference identifier |
| `internalBrandId` | `String` | Internal brand relationship identifier |

## Aggregate Functions

The object supports analytical operations including `count`, `countDistinct`, `has`, `sum`, `min`, `max`, `avg`, `collect`, `minDateTime`, and `maxDateTime` functions, each accepting a field name and optional conditions.

## Interfaces

The `BrandName` type implements the [`NodeFunctions`](/reference/graphql_api/interfaces/node-functions) interface, providing standard query capabilities.
