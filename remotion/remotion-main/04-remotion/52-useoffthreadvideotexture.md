"---\nimage: /generated/articles-docs-use-offthread-video-texture.png\nid: use-offthread-video-texture\ncrumb: '@remotion/three'\nsidebar_label: useOffthreadVideoTexture()\n---\n\n# ~useOffthreadVideoTexture()~<AvailableFrom v=\"4.0.83\"/>\n\n:::warning\nDeprecated - we recommend using [`@remotion/media`](/docs/videos/as-threejs-texture) instead.\n:::\n\nAllows you to use a video in React Three Fiber that is synchronized with Remotion's `useCurrentFrame()`, similar to [`useVideoTexture()`](/docs/use-video-texture), but uses [`<OffthreadVideo>`](/docs/offthreadvideo) instead.\n\nThis hook only works during rendering. In the Player and the Remotion Studio, use [`useVideoTexture()`](/docs/use-video-texture) instead. See below for an example.\n\nThis hook was designed to combat limitations of the default `<Html5Video>` element that is used with `useVideoTexture()` hook.\nSee: [Comparison of video tags](/docs/video-tags)\n\nThe return type of it is a `THREE.Texture | null` which you can assign as a `map` to for example `meshBasicMaterial`. We recommend to only render the material when the texture is not `null` to prevent bugs.\n\n## Example\n\n```tsx twoslash title=\"Simple usage (only works during rendering)\"\nimport {ThreeCanvas, useOffthreadVideoTexture} from '@remotion/three';\nimport {staticFile, useVideoConfig} from 'remotion';\n\nconst videoSrc = staticFile('/vid.mp4');\n\nconst My3DVideo = () => {\n  const {width, height} = useVideoConfig();\n\n  const videoTexture = useOffthreadVideoTexture({src: videoSrc});\n\n  return (\n    <ThreeCanvas width={width} height={height}>\n      <mesh>{videoTexture ? <meshBasicMaterial map={videoTexture} /> : null}</mesh>\n    </ThreeCanvas>\n  );\n};\n```\n\n```tsx twoslash title=\"Use useVideoTexture() only during rendering\"\nimport {ThreeCanvas, useOffthreadVideoTexture, useVideoTexture} from '@remotion/three';\nimport {useRef} from 'react';\nimport {staticFile, useRemotionEnvironment, useVideoConfig, Html5Video} from 'remotion';\n\nconst videoSrc = staticFile('/vid.mp4');\n\nconst useVideoOrOffthreadVideoTexture = (videoSrc: string, videoRef: React.RefObject<HTMLVideoElement | null>) => {\n  const env = useRemotionEnvironment();\n  if (env.isRendering) {\n    // eslint-disable-next-line react-hooks/rules-of-hooks\n    return useOffthreadVideoTexture({src: videoSrc});\n  }\n\n  // eslint-disable-next-line react-hooks/rules-of-hooks\n  return useVideoTexture(videoRef);\n};\n\nconst My3DVideo = () => {\n  const {width, height} = useVideoConfig();\n\n  const videoRef = useRef<HTMLVideoElement | null>(null);\n  const videoTexture = useVideoOrOffthreadVideoTexture(videoSrc, videoRef);\n  const env = useRemotionEnvironment();\n\n  return (\n    <>\n      {env.isRendering ? null : <Html5Video ref={videoRef} src={videoSrc} style={{position: 'absolute', opacity: 0}} />}\n      <ThreeCanvas width={width} height={height}>\n        <mesh>{videoTexture ? <meshBasicMaterial map={videoTexture} /> : null}</mesh>\n      </ThreeCanvas>\n    </>\n  );\n};\n```\n\n## API\n\nTakes an object with the following properties:\n\n### `src`\n\nThe video source. Can be a URL or a [`staticFile()`](/docs/staticfile).\n\n### `playbackRate`\n\nThe playback rate of the video. Defaults to `1`.\n\n:::note\nPlaying a video in reverse is not supported.\n:::\n\n### `transparent?`\n\nIf set to `true`, frames will be extracted as PNG, enabling transparency but also slowing down your render.\n\nIf set to `false` (_default_), frames will be extracted as bitmap (BMP), which is faster.\n\n### `toneMapped`<AvailableFrom v=\"4.0.117\" />\n\nSince Remotion 4.0.117, Remotion will adjust the colors of videos in different color spaces (such as HDR) when converting to RGB, to counteract color shifts.  \nSince the browser is painting in sRGB, this is necessary to ensure that the colors are displayed correctly.  \nThis behavior is by default `true` and can be disabled by setting `toneMapped` to `false`.  \nDisabling tone mapping will speed up rendering, but may result in less vibrant colors.\n\nPrior to Remotion 4.0.117, tone mapping was always disabled, and the `toneMapped` prop was not available.\n\n### `delayRenderTimeoutInMilliseconds`<AvailableFrom v=\"4.0.271\" />\n\nCustomize the [timeout](/docs/delay-render#modifying-the-timeout) of the [`delayRender()`](/docs/delay-render) call that this hook makes.\n\n### `delayRenderRetries`<AvailableFrom v=\"4.0.271\" />\n\nCustomize the [number of retries](/docs/delay-render#retrying) of the [`delayRender()`](/docs/delay-render) call that this hook makes.\n\n## Looping a texture\n\nLike `<OffthreadVideo>`, looping a video [is not supported out of the box](/docs/offthreadvideo#looping-a-offthreadvideo) but can be achieved with the `<Loop>` component.\n\n## See also\n\n- [Source code for this function](https://github.com/remotion-dev/remotion/blob/main/packages/three/src/use-offthread-video-texture.ts)\n- [`useVideoTexture()`](/docs/use-video-texture)\n- [`<ThreeCanvas />`](/docs/three-canvas)\n- [Comparison of video tags](/docs/video-tags)\n"
