/>\n\nThe \"Constant Rate Factor\" (CRF) of the output. [Use this setting to tell FFmpeg how to trade off size and quality.](/docs/encoding#controlling-quality-using-the-crf-setting)\n\nRanges for CRF scale, by codec:\n\n- `h264` crf range is 1-51 where crf 18 is _default_.\n- `h265` crf range is 0-51 where crf 23 is _default_.\n- `vp8` crf range is 4-63 where crf 9 is _default_.\n- `vp9` crf range is 0-63 where crf 28 is _default_.\n- `av1` crf range is 0-63 where crf 30 is _default_.\n\nThe lowest value is lossless, and the highest value is the worst quality possible. Higher values decrease the filesize at the cost of quality.\n\nThe range is exponential, so increasing the CRF value +6 results in roughly half the bitrate / file size, while -6 leads to roughly twice the bitrate.\n\nChoose the highest CRF value that still provides an acceptable quality. If the output looks good, then try a higher value.
