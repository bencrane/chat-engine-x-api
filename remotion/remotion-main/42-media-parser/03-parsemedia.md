"---\nimage: /generated/articles-docs-media-parser-parse-media.png\nid: parse-media\ntitle: parseMedia()\nslug: /media-parser/parse-media\ncrumb: '@remotion/media-parser'\n---\n\n:::warning\n[We are phasing out Media Parser and are moving to Mediabunny](/blog/mediabunny)!\n:::\n\n_Part of the [`@remotion/media-parser`](/docs/media-parser) package._\n_available from v4.0.190_\n\n## Examples\n\n```tsx twoslash title=\"Parsing a hosted video\"\nimport {parseMedia} from '@remotion/media-parser';\n\nconst result = await parseMedia({\n  src: 'https://remotion.media/video.mp4',\n  fields: {\n    durationInSeconds: true,\n    dimensions: true,\n  },\n});\n\nconsole.log(result.durationInSeconds); // 10\nconsole.log(result.dimensions); // {width: 1920, height: 1080}\n```\n\n```tsx twoslash title=\"Parsing a local file\"\nimport {parseMedia} from '@remotion/media-parser';\nimport {nodeReader} from '@remotion/media-parser/node';\n\nconst result = await parseMedia({\n  src: '/Users/jonnyburger/Downloads/video.mp4',\n  fields: {\n    durationInSeconds: true,\n    dimensions: true,\n  },\n  reader: nodeReader,\n});\n```\n\n## API\n\n:::warning\n**Unstable API**: This package is experimental. The API may change in the future.  \nThe API for getting video metadata is stable and may be used in production.\n:::\n\n### `src`\n\nEither a local file path, or a URL, or a `File` or `Blob` object.\n\nIf you pass a local file path, you must also pass [`nodeReader`](/docs/media-parser/node-reader) as the `reader` argument.\n\n### `fields?`\n\nAn object specifying which fields you'd like to receive.  \nIf you like to receive the field, pass `true` as the value.\n\nSee [Available Fields](/docs/media-parser/fields) for a list of available fields.\n\n### `reader?`\n\nA [reader](/docs/media-parser/readers) interface.\n\nIf you pass [`nodeReader`](/docs/media-parser/node-reader), you must pass a local file path as the `src` argument.\n\n### `controller?`\n\nA [controller](/docs/media-parser/media-parser-controller) object that allows you to pause, resume and abort the parsing process.\n\n### `onVideoTrack?`\n\n[A callback that is called when a video track is detected.](/docs/media-parser/samples)  \nIt receives an object with:\n\n- `track` (of type [`MediaParserVideoTrack`](/docs/media-parser/types#mediaparservideotrack))\n- `container` (of type [`MediaParserContainer`](/docs/media-parser/types#mediaparsercontainer))\n\nYou must return either `null` or a callback that is called for each sample that corresponds to the video track.\n\nThe `sample` has the type [`MediaParserVideoSample`](/docs/media-parser/types#mediaparservideosample), which is compatible with the mandatory fields for the WebCodecs [`EncodedVideoChunk`](https://developer.mozilla.org/en-US/docs/Web/API/EncodedVideoChunk) constructor.\n\n```tsx twoslash title=\"Reading video frames\"\nimport {parseMedia, MediaParserOnVideoTrack} from '@remotion/media-parser';\n\nconst onVideoTrack: MediaParserOnVideoTrack = ({track}) => {\n  console.log(track);\n\n  return (sample) => {\n    console.log(new EncodedVideoChunk(sample));\n  };\n};\n```\n\n### `onAudioTrack?`\n\n[A callback that is called when an audio track is detected.](/docs/media-parser/samples)  \nIt receives an object with:\n\n- `track` (of type [`MediaParserAudioTrack`](/docs/media-parser/types#mediaparseraudiotrack))\n- `container` (of type [`MediaParserContainer`](/docs/media-parser/types#mediaparsercontainer))\n\nYou must return either `null` or a callback that is called for each sample that corresponds to the audio track.\n\nThe `sample` has the type [`MediaParserAudioSample`](/docs/media-parser/types#mediaparseraudiosample), which is compatible with the mandatory fields for the WebCodecs [`EncodedAudioChunk`](https://developer.mozilla.org/en-US/docs/Web/API/EncodedAudioChunk) constructor.\n\n```tsx twoslash title=\"Reading audio frames\"\n// @noErrors\nimport {parseMedia, MediaParserOnAudioTrack} from '@remotion/media-parser';\n\nconst onAudioTrack: MediaParserOnAudioTrack = ({track}) => {\n  console.log(track);\n\n  return (sample) => {\n    console.log(new EncodedAudioChunk(sample));\n  };\n};\n```\n\n### `selectM3uStream?`\n\nA callback that is called when a `.m3u8` file is detected which has multiple streams.  \nSee [Stream selection](/docs/media-parser/stream-selection) for an example.\n\n### `selectM3uAssociatedPlaylists?`\n\nA callback that is called after a M3U stream has been selected, allowing you to pick which auxiliary playlists should be parsed. Usually this is used for selecting an audio track.  \nSee [Stream selection](/docs/media-parser/stream-selection) for an example.\n\n### `onParseProgress?`\n\nA callback that is called from time to time when bytes have been read from the media.  \nIt includes the following data:\n\n```tsx twoslash\nimport {ParseMediaProgress} from '@remotion/media-parser';\n//          ^?\n```\n\n:::note\nYou may make this an async function, and while it is not resolved, **the parsing process will be paused**.  \nThis is useful if you want to add a small delay inbetween progress steps to keep the UI interactive.\n:::\n\n### `progressIntervalInMs?`\n\n_number_\n\nThe interval in milliseconds at which the [`onParseProgress`](#onparseprogress) callback is called.  \nDefault `100`. Set to `0` for unthrottled and synchronous updates.  \nNote that updates are fired very often and updating the DOM often may slow down the conversion process.\n\n### `makeSamplesStartAtZero`\n\n_boolean_\n\nIf this is set to true, the timestamps of the samples returned will be recalculated so they start at 0 instead of the timestamps that are contained in the file. Default `true`.\n\n### `seekingHints?`\n\nAn object that contains hints about the structure of the media file.\n\nSee [Seeking Hints](/docs/media-parser/seeking-hints) for more information.\n\n### `logLevel?`\n\nOne of `\"error\"`, `\"warn\"`, `\"info\"`, `\"debug\"`, `\"trace\"`.  \nDefault value: `\"info\"`, which logs only important information.\n\n### `acknowledgeRemotionLicense?`\n\nAcknowledge the [Remotion License](https://remotion.dev/license) to make the console message disappear.\n\n## Callbacks\n\nEach field also has a callback that allows you to retrieve the value as soon as it is obtained without waiting for the function to resolve.\n\nYou do not have to add the field to the [`fields`](#fields) object if you use the callback.  \nHowever, just like with [`fields`](#fields), adding a callback for a [slow field](/docs/media-parser/fast-and-slow) may require reading more of the file.\n\n```tsx twoslash title=\"Using a callback\"\nimport {parseMedia} from '@remotion/media-parser';\n\nconst result = await parseMedia({\n  src: 'https://remotion.media/video.mp4',\n  onDurationInSeconds: (durationInSeconds) => {\n    console.log(durationInSeconds);\n  },\n  onDimensions: (dimensions) => {\n    console.log(dimensions);\n  },\n});\n```\n\n## See also\n\n- [Source code for this function](https://github.com/remotion-dev/remotion/blob/main/packages/media-parser/src/parse-media.ts)\n- [`@remotion/media-parser`](/docs/media-parser)\n"
