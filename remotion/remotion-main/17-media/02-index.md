"---\nimage: /generated/articles-docs-media-video.png\ntitle: '<Video>'\ncrumb: 'API - @remotion/media'\n---\n\n:::note\nThis is documentation for the new `<Video>` tag.  \nNot to be confused with the older [`<Html5Video>` / `<Video>`](/docs/html5-video) tag from `remotion`.\n:::\n\nThis component imports and displays a video, similar to [`<OffthreadVideo/>`](/docs/offthreadvideo) but during rendering, extracts the exact frame from the video using [Mediabunny](/docs/mediabunny) and displays it in a `<canvas>` tag.\n\nThis component has [native buffering support](/docs/player/buffer-state) enabled by default. When used in the Player, it automatically pauses playback when buffering and resumes when ready.\n\n## Example\n\n```tsx twoslash\nimport {AbsoluteFill, staticFile} from 'remotion';\nimport {Video} from '@remotion/media';\n\nexport const MyVideo = () => {\n  return (\n    <AbsoluteFill>\n      <Video src={staticFile('video.webm')} />\n    </AbsoluteFill>\n  );\n};\n```\n\nYou can load a video from an URL as well:\n\n```tsx twoslash\nimport {AbsoluteFill} from 'remotion';\nimport {Video} from '@remotion/media';\n// ---cut---\nexport const MyComposition = () => {\n  return (\n    <AbsoluteFill>\n      <Video src=\"https://remotion.media/BigBuckBunny.mp4\" />\n    </AbsoluteFill>\n  );\n};\n```\n\n## Props\n\n### `src`\n\nThe URL of the video to be rendered. Can be a remote URL or a local file referenced with [`staticFile()`](/docs/staticfile).\n\n### `trimBefore?`\n\nWill remove a portion of the video at the beginning (left side).\n\nIn the following example, we assume that the [`fps`](/docs/composition#fps) of the composition is `30`.\n\nBy passing `trimBefore={60}`, the playback starts immediately, but with the first 2 seconds of the video trimmed away.  \nBy passing `trimAfter={120}`, any video after the 4 second mark in the file will be trimmed away.\n\nThe video will play the range from `00:02:00` to `00:04:00`, meaning the video will play for 2 seconds.\n\nFor exact behavior, see: [Order of operations](/docs/audio/order-of-operations).\n\n```tsx twoslash\nimport {AbsoluteFill, staticFile} from 'remotion';\nimport {Video} from '@remotion/media';\n\n// ---cut---\nexport const MyComposition = () => {\n  return (\n    <AbsoluteFill>\n      <Video src={staticFile('video.webm')} trimBefore={60} trimAfter={120} />\n    </AbsoluteFill>\n  );\n};\n```\n\n### `trimAfter?`\n\nRemoves a portion of the video at the end (right side). See [`trimBefore`](#trimbefore) for an explanation.\n\n### `volume?`\n\nAllows you to control the volume of the audio in it's entirety or frame by frame.  \nRead the page on [using audio](/docs/using-audio) to learn more.\n\n```tsx twoslash title=\"Setting a static volume\"\nimport {AbsoluteFill, staticFile} from 'remotion';\nimport {Video} from '@remotion/media';\n\nexport const MyVideo = () => {\n  return (\n    <AbsoluteFill>\n      <Video volume={0.5} src={staticFile('video.webm')} />\n    </AbsoluteFill>\n  );\n};\n```\n\n```tsx twoslash title=\"Changing the volume over time\"\nimport {AbsoluteFill, interpolate, staticFile} from 'remotion';\nimport {Video} from '@remotion/media';\n\nexport const MyVideo = () => {\n  return (\n    <AbsoluteFill>\n      <Video volume={(f) => interpolate(f, [0, 30], [0, 1], {extrapolateLeft: 'clamp', extrapolateRight: 'clamp'})} src={'https://remotion.media/video.mp4'} />\n    </AbsoluteFill>\n  );\n};\n```\n\n### `name?`\n\nA name and that will be shown as the label of the sequence in the timeline of the Remotion Studio. This property is purely for helping you keep track of items in the timeline.\n\n### `onError?`<AvailableFrom v=\"4.0.404\" />\n\nHandle errors that occur during video processing. The callback receives an `Error` property and should return either `'fallback'` or `'fail'`.\n\n- Return `'fallback'` to fall back to [`<OffthreadVideo>`](/docs/offthreadvideo) (default behavior)\n- Return `'fail'` to fail the render immediately\n\n```tsx twoslash\nimport {staticFile} from 'remotion';\nimport {Video} from '@remotion/media';\n\nexport const MyVideo = () => {\n  return (\n    <Video\n      src={'https://remotion.media/video.mp4'}\n      onError={(error) => {\n        console.log('Video error:', error.message);\n\n        // Return 'fail' to fail the render, or 'fallback' to use <OffthreadVideo>\n        return 'fallback';\n      }}\n    />\n  );\n};\n```\n\n:::note\n\n- In client-side rendering, the render will always fail regardless of the return value.\n- If you return `'fallback'`, the component will fall back to [`<OffthreadVideo>`](/docs/offthreadvideo) which may have its own error handling via [`fallbackOffthreadVideoProps.onError`](#onerror-1).\n  :::\n\n### `playbackRate?`<AvailableFrom v=\"4.0.354\" />\n\nControls the speed of the video. `1` is the default and means regular speed, `0.5` slows down the video so it's twice as long and `2` speeds up the video so it's twice as fast.\n\n```tsx twoslash title=\"Example of a video playing twice as fast\"\nimport {AbsoluteFill, staticFile} from 'remotion';\nimport {Video} from '@remotion/media';\n\n// ---cut---\nexport const MyComposition = () => {\n  return (\n    <AbsoluteFill>\n      <Video playbackRate={2} src={'https://remotion.media/video.mp4'} />\n    </AbsoluteFill>\n  );\n};\n```\n\n:::note\nPlaying a video in reverse is not supported.\n:::\n\n### `muted?`\n\nYou can drop the audio of the video by adding a `muted` prop:\n\n```tsx twoslash title=\"Example of a muted video\"\nimport {AbsoluteFill, OffthreadVideo} from 'remotion';\nimport {Video} from '@remotion/media';\n// ---cut---\nexport const MyComposition = () => {\n  return (\n    <AbsoluteFill>\n      <Video muted src=\"https://remotion.media/BigBuckBunny.mp4\" />\n    </AbsoluteFill>\n  );\n};\n```\n\n### `style?`\n\nYou can pass any style you can pass to a native HTML `<canvas>` element.\n\n```tsx twoslash\nimport {AbsoluteFill, staticFile} from 'remotion';\nimport {Video} from '@remotion/media';\n\n// ---cut---\nexport const MyComposition = () => {\n  return (\n    <AbsoluteFill>\n      <Video src={staticFile('video.webm')} style={{height: 720, width: 1280}} />\n    </AbsoluteFill>\n  );\n};\n```\n\n### `loop?`\n\nMakes the video loop indefinitely.\n\n```tsx twoslash title=\"Example of a looped video\"\nimport {AbsoluteFill} from 'remotion';\nimport {Video} from '@remotion/media';\n\n// ---cut---\nexport const MyComposition = () => {\n  return (\n    <AbsoluteFill>\n      <Video loop src=\"https://remotion.media/BigBuckBunny.mp4\" />\n    </AbsoluteFill>\n  );\n};\n```\n\n:::note\nWhen a video ends (and `loop` is not set), the last frame of the video remains visible by default.  \nThis matches the behavior of [`<Html5Video>`](/docs/html5-video#loop).\n:::\n\n### `loopVolumeCurveBehavior?`<AvailableFrom v=\"4.0.354\" />\n\nControls the `frame` which is returned when using the [`volume`](#volume) callback function and adding the [`loop`](#loop) attribute.\n\nCan be either `\"repeat\"` (default, start from 0 on each iteration) or `\"extend\"` (keep increasing frames).\n\n### `showInTimeline?`\n\nIf set to `false`, no layer will be shown in the timeline of the Remotion Studio. The default is `true`.\n\n### `delayRenderTimeoutInMilliseconds?`\n\nCustomize the [timeout](/docs/delay-render#modifying-the-timeout) of the [`delayRender()`](/docs/delay-render) call that this component makes.\n\n### `delayRenderRetries?`\n\nCustomize the [number of retries](/docs/delay-render#retrying) of the [`delayRender()`](/docs/delay-render) call that this component makes.\n\n### `onVideoFrame?`\n\nA callback function that gets called when a frame is extracted from the video.  \nUseful for [video manipulation](/docs/video-manipulation).  \nThe callback is called with a [`CanvasImageSource`](https://developer.mozilla.org/en-US/docs/Web/API/CanvasImageSource) object, more specifically, either an `ImageBitmap` or a `VideoFrame`.\n\n### `audioStreamIndex?`\n\nSelect the audio stream to use. The default is `0`.\n\n```tsx twoslash\nimport {AbsoluteFill} from 'remotion';\nimport {Video} from '@remotion/media';\n\n// ---cut---\nexport const MyComposition = () => {\n  return (\n    <AbsoluteFill>\n      <Video audioStreamIndex={1} src={'https://remotion.media/multiple-audio-streams.mov'} />\n    </AbsoluteFill>\n  );\n};\n```\n\n### `credentials?`<AvailableFrom v=\"4.0.437\" />\n\nControls the [`credentials`](https://developer.mozilla.org/en-US/docs/Web/API/Request/credentials) option of the `fetch()` requests made to retrieve the video data.\n\nAccepts `\"omit\"`, `\"same-origin\"` (default behavior of `fetch()`) or `\"include\"`.\nSet to `\"include\"` if you need to send cookies or authentication headers to a cross-origin video URL.\n\n```tsx twoslash\nimport {AbsoluteFill} from 'remotion';\nimport {Video} from '@remotion/media';\n\n// ---cut---\nexport const MyComposition = () => {\n  return (\n    <AbsoluteFill>\n      <Video credentials=\"include\" src=\"https://example.com/protected-video.mp4\" />\n    </AbsoluteFill>\n  );\n};\n```\n\n### `toneFrequency?`\n\nAccepts a number between `0.01` and `2`, where `1` represents the original pitch. Values less than `1` will decrease the pitch, while values greater than `1` will increase it.\n\nA `toneFrequency` of 0.5 would lower the pitch by half, and a `toneFrequency` of `1.5` would increase the pitch by 50%.\n\nThe value must stay the same across the entire time the video tag is mounted.\n\n:::warning\nOnly works in server-side rendering.  \nDoes not currently work in preview or in [client-side rendering](/docs/client-side-rendering/).\n:::\n\n### `headless?`<AvailableFrom v=\"4.0.387\" />\n\nDoes not mount a `<canvas>`, but still allows you to use the [`onVideoFrame`](#onvideoframe) prop.  \nThis is useful for [embedding a video as a texture in Three.js](/docs/videos/as-threejs-texture).\n\n### `fallbackOffthreadVideoProps?`\n\nWhen a [fallback to `<OffthreadVideo>`](/docs/media/fallback) happens, this prop allows you to pass props to the fallback `<OffthreadVideo>` component.  \nOnly props that are not supported by `<Video>` from `@remotion/media` need to be specified here - props that apply for both tags will automatically be forwarded and do not need to be specified here.\n\n:::note\nThis prop has no effect when using [`@remotion/web-renderer`](/docs/web-renderer) for client-side rendering, as fallback is not possible. See [Fallback is not possible in client-side rendering](/docs/media/fallback#fallback-is-not-possible-in-client-side-rendering).\n:::\n\n#### `acceptableTimeShiftInSeconds?`\n\nMaps to [`<OffthreadVideo />` -> `acceptableTimeShiftInSeconds`](/docs/offthreadvideo#acceptabletimeshiftinseconds)\n\n#### `transparent?`\n\nMaps to [`<OffthreadVideo />` -> `transparent`](/docs/offthreadvideo#transparent)\n\n#### `toneMapped?`\n\nMaps to [`<OffthreadVideo />` -> `toneMapped`](/docs/offthreadvideo#tonemapped)\n\n#### `onError?`\n\nMaps to [`<OffthreadVideo />` -> `onError`](/docs/offthreadvideo#onerror)\n\n#### `crossOrigin?`\n\nMaps to [`<OffthreadVideo />` -> `crossOrigin`](/docs/offthreadvideo#crossorigin)\n\n#### `useWebAudioApi?`\n\nMaps to [`<OffthreadVideo />` -> `useWebAudioApi`](/docs/offthreadvideo#usewebaudioapi)\n\n#### `pauseWhenBuffering?`\n\nMaps to [`<OffthreadVideo />` -> `pauseWhenBuffering`](/docs/offthreadvideo#pausewhenbuffering)\n\n#### `onAutoPlayError?`\n\nMaps to [`<OffthreadVideo />` -> `onAutoPlayError`](/docs/offthreadvideo#onautoplayerror)\n\n### `disallowFallbackToOffthreadVideo?`\n\nBy default, if the video cannot be embedded using this tag, [a fallback to `<OffthreadVideo>`](/docs/media/fallback) will be attempted.\n\nPass this prop to disable the fallback and fail the render instead.\n\n:::note\nWhen using [`@remotion/web-renderer`](/docs/web-renderer) for client-side rendering, fallback is not possible and the render will always fail if the video cannot be embedded. See [Fallback is not possible in client-side rendering](/docs/media/fallback#fallback-is-not-possible-in-client-side-rendering).\n:::\n\n### `debugOverlay?`\n\nShows a debug overlay on the video. This is useful for debugging the video playback.\n\n### `debugAudioScheduling?`\n\nLogs audio scheduling information to the console. This is useful for debugging audio playback timing issues.\n\n## See also\n\n- [Source code for this component](https://github.com/remotion-dev/remotion/blob/main/packages/media/src/video/video.tsx)\n- [Comparison of video tags](/docs/video-tags)\n- [`<Audio>` (from `@remotion/media`)](/docs/media/audio)\n- [`<Html5Video>` (from `remotion`)](/docs/html5-video)\n- [`<OffthreadVideo>`](/docs/offthreadvideo)\n"
