"---\nimage: /generated/articles-docs-renderer-get-video-metadata.png\nid: get-video-metadata\ntitle: getVideoMetadata()\ncrumb: '@remotion/renderer'\n---\n\n# getVideoMetadata()<AvailableFrom v=\"4.0.6\" />\n\n:::note\nDeprecated.  \nBetter solution: [Get metadata using Mediabunny](/docs/mediabunny/metadata)\n:::\n\n:::note\nOnly works in Node.js and Bun.\n:::\n\nGets metadata about a video file in Node.js. Useful for calculating metadata on a server.\n\n## Example\n\n```ts twoslash\nimport {getVideoMetadata, VideoMetadata} from '@remotion/renderer';\n\nconst videoMetadata: VideoMetadata = await getVideoMetadata('/Users/john/Documents/bunny.mp4');\n\nconst {width, height, fps, durationInSeconds, codec, supportsSeeking, colorSpace, audioCodec, audioFileExtension, pixelFormat} = videoMetadata;\n```\n\n:::info\nPass an absolute path to `getVideoMetadata()`. URLs are not supported.\n:::\n\n## Arguments\n\n### `videoSource`\n\n_string_\n\nA local video file path.\n\n### options?\n\n### `logLevel?`\n\n<Options id=\"log\" />\n\n#### `binariesDirectory?`<AvailableFrom v=\"4.0.120\" />\n\n<Options id=\"binaries-directory\" />\n\n## Return Value\n\nThe return value is an object with the following properties:\n\n### `fps`\n\n_number_\n\nThe amount of frames per seconds the video has (framerate)\n\n### `width`\n\n_number_\n\nThe width of the video in px.\n\n### `height`\n\n_number_\n\nThe height of the video in px.\n\n### `durationInSeconds`\n\n_number_\n\nThe time length of the video in seconds.\n\n### `codec`<AvailableFrom v=\"4.0.8\" />\n\n_string_\n\nOne of `'h264' | 'h265' | 'vp8' | 'vp9' | 'av1' | 'prores'` or ` 'unknown'` if the codec is not officially supported by Remotion.\n\n### `supportsSeeking`<AvailableFrom v=\"4.0.8\" />\n\n_boolean_\n\nA prediction whether the video will be seekable in the browser. The algorithm is:\n\n1. If the codec is `unknown`, then the video is not seekable (`false`).\n2. If the video is shorter than 5 seconds, then it is seekable (`true`).\n3. If the codec is not `h264`, then it is seekable (`true`).\n4. The codec is now `h264`. If it supports Faststart (the `moov` atom is in front of the `mdat` atom), then it is seekable (`true`)\n5. Otherwise, it is not seekable (`false`).\n\nIf a video is not seekable, you might run into the [\"Non-seekable media\"](/docs/non-seekable-media) error.  \nThis means that the video might fail to render if embedded in a `<Html5Video>` tag and be slow if embedded in a `<OffthreadVideo>` tag.\n\nYou may consider re-encoding the video using FFmpeg to make it seekable.\n\n### `colorSpace`<AvailableFrom v=\"4.0.28\"/>\n\nOne of `rgb`, `bt601`, `bt709`, `bt2020-ncl`, `bt2020-cl`, `fcc`, `bt470bg`, `smpte170m`, `smpte240m`, `ycgco`, `smpte2085`, `chroma-derived-ncl`, `chroma-derived-cl`, `ictcp` or `unknown`.\n\n### `audioCodec`<AvailableFrom v=\"4.0.49\"/>\n\nIf the video has no audio track, it is `null`.\nIf the audio codec is unknown to Remotion, it is `\"unknown\"`.\n\nOtherwise, it is one of `\"opus\" | \"aac\" | \"mp3\" | \"pcm-f16le\" | \"pcm-f24le\" | \"pcm-f32be\" | \"pcm-s16be\" | \"pcm-s16le\" | \"pcm-f32le\" | \"pcm-s32be\" | \"pcm-s32le\" | \"pcm-s64be\" | \"pcm-s64le\" | \"pcm-u16be\" | \"pcm-u16le\" | \"pcm-u24be\" | \"pcm-u24le\" | \"pcm-u32be\" | \"pcm-u32le\" | \"pcm-u8\" | \"pcm-f64be\" | \"pcm-s24be\" | \"pcm-s24le\" | \"pcm-s8\" | \"pcm-s16be-planar\" | \"pcm-s8-planar\" | \"pcm-s24le-planar\" | \"pcm-s32le-planar\" | \"unknown\"`\n\n### `audioFileExtension`<AvailableFrom v=\"4.0.49\"/>\n\nIf the video has no audio track or is unknown to Remotion, it is `null`. Otherwise it is the appropriate file extension for the audio codec, e.g. `\"mp3\"`.
