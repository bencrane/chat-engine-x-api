"---\nimage: /generated/articles-docs-media-audio.png\ntitle: '<Audio>'\ncrumb: 'API - @remotion/media'\n---\n\n:::note\nThis is documentation for the new `<Audio>` tag.  \nNot to be confused with the older [`<Html5Audio>` / `<Audio>`](/docs/html5-audio) tag from `remotion`.\n:::\n\nThis component imports and plays audio, similar to [`<Html5Audio />`](/docs/html5-audio) from `remotion` but during rendering, extracts the exact audio using [Mediabunny](/docs/mediabunny) instead of FFmpeg.\n\n## Example\n\n```tsx twoslash\nimport {AbsoluteFill, staticFile} from 'remotion';\nimport {Audio} from '@remotion/media';\n\nexport const MyVideo = () => {\n  return (\n    <AbsoluteFill>\n      <Audio src={staticFile('audio.mp3')} />\n    </AbsoluteFill>\n  );\n};\n```\n\nYou can load a video from an URL as well:\n\n```tsx twoslash\nimport {AbsoluteFill} from 'remotion';\nimport {Audio} from '@remotion/media';\n// ---cut---\nexport const MyComposition = () => {\n  return (\n    <AbsoluteFill>\n      <Audio src=\"https://remotion.media/audio.wav\" />\n    </AbsoluteFill>\n  );\n};\n```\n\n## Props\n\n### `src`\n\nThe URL of the audio to be rendered. Can be a remote URL or a local file referenced with [`staticFile()`](/docs/staticfile).\n\n### `trimBefore?`\n\nWill remove a portion of the audio at the beginning (left side).\n\nIn the following example, we assume that the [`fps`](/docs/composition#fps) of the composition is `30`.\n\nBy passing `trimBefore={60}`, the playback starts immediately, but with the first 2 seconds of the audio trimmed away.  \nBy passing `trimAfter={120}`, any audio after the 4 second mark in the file will be trimmed away.\n\nThe audio will play the range from `00:02:00` to `00:04:00`, meaning the audio will play for 2 seconds.\n\nFor exact behavior, see: [Order of operations](/docs/audio/order-of-operations).\n\n```tsx twoslash\nimport {AbsoluteFill, staticFile} from 'remotion';\nimport {Audio} from '@remotion/media';\n\n// ---cut---\nexport const MyComposition = () => {\n  return (\n    <AbsoluteFill>\n      <Audio src={staticFile('audio.mp3')} trimBefore={60} trimAfter={120} />\n    </AbsoluteFill>\n  );\n};\n```\n\n### `trimAfter?`\n\nRemoves a portion of the audio at the end (right side). See [`trimBefore`](#trimbefore) for an explanation.\n\n### `volume?`\n\nAllows you to control the volume of the audio in it's entirety or frame by frame.  \nRead the page on [using audio](/docs/using-audio) to learn more.\n\n```tsx twoslash title=\"Setting a static volume\"\nimport {AbsoluteFill, staticFile} from 'remotion';\nimport {Audio} from '@remotion/media';\n\nexport const MyVideo = () => {\n  return (\n    <AbsoluteFill>\n      <Audio volume={0.5} src={staticFile('background.mp3')} />\n    </AbsoluteFill>\n  );\n};\n```\n\n```tsx twoslash title=\"Changing the volume over time\"\nimport {AbsoluteFill, interpolate, staticFile} from 'remotion';\nimport {Audio} from '@remotion/media';\n\nexport const MyVideo = () => {\n  return (\n    <AbsoluteFill>\n      <Audio volume={(f) => interpolate(f, [0, 30], [0, 1], {extrapolateLeft: 'clamp'})} src={staticFile('voice.mp3')} />\n    </AbsoluteFill>\n  );\n};\n```\n\n### `name?`\n\nA name and that will be shown as the label of the sequence in the timeline of the Remotion Studio. This property is purely for helping you keep track of items in the timeline.\n\n### `onError?`<AvailableFrom v=\"4.0.404\" />\n\nHandle errors that occur during audio processing. The callback receives an `Error` and should return either `'fallback'` or `'fail'`.\n\n- Return `'fallback'` to fall back to [`<Html5Audio>`](/docs/html5-audio) (default behavior)\n- Return `'fail'` to fail the render immediately\n\n```tsx twoslash\nimport {Audio} from '@remotion/media';\n\nexport const MyVideo = () => {\n  return (\n    <Audio\n      src={'https://remotion.media/audio.mp3'}\n      onError={(error) => {\n        console.log('Audio error:', error.message);\n\n        // Return 'fail' to fail the render, or 'fallback' to use <Html5Audio>\n        return 'fallback';\n      }}\n    />\n  );\n};\n```\n\n:::note\n\n- In client-side rendering, the render will always fail regardless of the return value.\n- If you return `'fallback'`, the component will fall back to [`<Html5Audio>`](/docs/html5-audio) which may have its own error handling via [`fallbackHtml5AudioProps.onError`](#onerror-1).\n  :::\n\n### `playbackRate?`\n\nControls the speed of the audio. `1` is the default and means regular speed, `0.5` slows down the audio so it's twice as long and `2` speeds up the audio so it's twice as fast.\n\n```tsx twoslash title=\"Example of a audio playing twice as fast\"\nimport {AbsoluteFill, staticFile} from 'remotion';\nimport {Audio} from '@remotion/media';\n\n// ---cut---\nexport const MyComposition = () => {\n  return (\n    <AbsoluteFill>\n      <Audio playbackRate={2} src={'https://remotion.media/audio.mp3'} />\n    </AbsoluteFill>\n  );\n};\n```\n\n:::note\nPlaying a audio in reverse is not supported.\n:::\n\n### `loop?`<AvailableFrom v=\"3.2.29\" />\n\nMakes the audio loop indefinitely.\n\n```tsx twoslash title=\"Example of a looped audio\"\nimport {Audio} from '@remotion/media';\n\n// ---cut---\nexport const MyComposition = () => {\n  return <Audio loop src=\"https://remotion.media/audio.wav\" />;\n};\n```\n\n### `loopVolumeCurveBehavior?`\n\nControls the `frame` which is returned when using the [`volume`](#volume) callback function and adding the [`loop`](#loop) attribute.\n\nCan be either `\"repeat\"` (default, start from 0 on each iteration) or `\"extend\"` (keep increasing frames).\n\n### `muted?`\n\nThe `muted` prop will be respected. It will lead to no audio being played while still keeping the audio tag mounted. It's value may change over time, for example to only mute a certain section of the audio.\n\n```tsx twoslash title=\"Example of a muted video\"\nimport {Audio} from '@remotion/media';\n// ---cut---\nexport const MyComposition = () => {\n  return <Audio muted src=\"https://remotion.media/audio.wav\" />;\n};\n```\n\n### `showInTimeline?`\n\nIf set to `false`, no layer will be shown in the timeline of the Remotion Studio. The default is `true`.\n\n### `delayRenderTimeoutInMilliseconds?`\n\nCustomize the [timeout](/docs/delay-render#modifying-the-timeout) of the [`delayRender()`](/docs/delay-render) call that this component makes.\n\n### `delayRenderRetries?`\n\nCustomize the [number of retries](/docs/delay-render#retrying) of the [`delayRender()`](/docs/delay-render) call that this component makes.\n\n### `audioStreamIndex?`\n\nSelect the audio stream to use. The default is `0`.\n\n```tsx twoslash\nimport {AbsoluteFill} from 'remotion';\nimport {Audio} from '@remotion/media';\n\n// ---cut---\nexport const MyComposition = () => {\n  return (\n    <AbsoluteFill>\n      <Audio audioStreamIndex={1} src={'https://remotion.media/multiple-audio-streams.mov'} />\n    </AbsoluteFill>\n  );\n};\n```\n\n### `credentials?`<AvailableFrom v=\"4.0.437\" />\n\nControls the [`credentials`](https://developer.mozilla.org/en-US/docs/Web/API/Request/credentials) option of the `fetch()` requests made to retrieve the audio data.\n\nAccepts `\"omit\"`, `\"same-origin\"` (default behavior of `fetch()`) or `\"include\"`.\nSet to `\"include\"` if you need to send cookies or authentication headers to a cross-origin audio URL.\n\n```tsx twoslash\nimport {AbsoluteFill} from 'remotion';\nimport {Audio} from '@remotion/media';\n\n// ---cut---\nexport const MyComposition = () => {\n  return (\n    <AbsoluteFill>\n      <Audio credentials=\"include\" src=\"https://example.com/protected-audio.mp3\" />\n    </AbsoluteFill>\n  );\n};\n```\n\n### `toneFrequency?`\n\nAccepts a number between `0.01` and `2`, where `1` represents the original pitch. Values less than `1` will decrease the pitch, while values greater than `1` will increase it.\n\nA `toneFrequency` of 0.5 would lower the pitch by half, and a `toneFrequency` of `1.5` would increase the pitch by 50%.\n\nThe value must stay the same across the entire time the video tag is mounted.\n\n:::warning\nOnly works in server-side rendering.  \nDoes not currently work in preview or in [client-side rendering](/docs/client-side-rendering/).\n:::\n\n### `fallbackHtml5AudioProps?`\n\nWhen a [fallback to `<Html5Audio>` from `remotion`](/docs/media/fallback) happens, this prop allows you to pass props to the fallback `<Html5Audio>` component.  \nOnly props that are not supported by `<Audio>` from `@remotion/media` need to be specified here - props that apply for both tags will automatically be forwarded and do not need to be specified here.\n\n:::note\nThis prop has no effect when using [`@remotion/web-renderer`](/docs/web-renderer) for client-side rendering, as fallback is not possible. See [Fallback is not possible in client-side rendering](/docs/media/fallback#fallback-is-not-possible-in-client-side-rendering).\n:::\n\n#### `onError?`\n\nMaps to [`<Html5Audio />` -> `onError`](/docs/html5-audio#onerror)\n\n#### `useWebAudioApi?`\n\nMaps to [`<Html5Audio />` -> `useWebAudioApi`](/docs/html5-audio#usewebaudioapi)\n\n#### `acceptableTimeShiftInSeconds?`\n\nMaps to [`<Html5Audio />` -> `acceptableTimeShiftInSeconds`](/docs/html5-audio#acceptabletimeshiftinseconds)\n\n#### `pauseWhenBuffering?`\n\nMaps to [`<Html5Audio />` -> `pauseWhenBuffering`](/docs/html5-audio#pausewhenbuffering)\n\n#### `crossOrigin?`\n\nMaps to [`<Html5Audio />` -> `crossOrigin`](/docs/html5-audio#crossorigin)\n\n### `disallowFallbackToHtml5Audio?`\n\nBy default, if the audio cannot be embedded using this tag, [a fallback to `<Html5Audio>` from `remotion`](/docs/media/fallback) will be attempted.\n\nPass this prop to disable the fallback and fail the render instead.\n\n:::note\nWhen using [`@remotion/web-renderer`](/docs/web-renderer) for client-side rendering, fallback is not possible and the render will always fail if the audio cannot be embedded. See [Fallback is not possible in client-side rendering](/docs/media/fallback#fallback-is-not-possible-in-client-side-rendering).\n:::\n\n### `debugAudioScheduling?`\n\nLogs audio scheduling information to the console. This is useful for debugging audio playback timing issues.\n\n## See also\n\n- [Source code for this component](https://github.com/remotion-dev/remotion/blob/main/packages/media/src/audio/audio.tsx)\n- [`<Html5Audio />` (from `remotion`)](/docs/html5-audio)\n- [`<Video />` (from `@remotion/media`)](/docs/media/video)\n- [`<OffthreadVideo>`](/docs/offthreadvideo)\n"
