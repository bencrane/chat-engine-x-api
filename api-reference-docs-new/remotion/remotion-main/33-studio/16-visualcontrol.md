"---\nimage: /generated/articles-docs-studio-visual-control.png\ntitle: visualControl()\ncrumb: '@remotion/studio'\n---\n\n# visualControl()<AvailableFrom v=\"4.0.292\" />\n\n:::warning\n**Preview feature:** A few issues are known, and filed under this [issue](https://github.com/remotion-dev/remotion/issues/5210).\n:::\n\nCreates a control in the right sidebar of the [Remotion Studio](/docs/studio) that allows you to change a value.\n\nUseful for if you have a hardcoded constant that you want to quickly find the optimal value for.\n\n## Example\n\nConsider you have a value `rotation` that you want to find the optimal value for:\n\n```tsx twoslash title=\"A simple composition\"\nimport {useVideoConfig} from 'remotion';\n\nconst MyComp: React.FC = () => {\n  const rotation = 180;\n\n  return <div style={{rotate: `${rotation}deg`}}>Hello</div>;\n};\n```\n\nInstead of changing the value of `rotation` in the code, you can create a control in the right sidebar of the [Remotion Studio](/docs/studio) that allows you to change the value:\n\n```tsx twoslash title=\"Creating a visual control with the name 'rotation'\"\nimport {visualControl} from '@remotion/studio';\n\nconst MyComp: React.FC = () => {\n  const rotation = visualControl('rotation', 180);\n\n  return <div style={{rotate: `${rotation}deg`}}>Hello</div>;\n};\n```\n\nNow, in the right sidebar of the [Remotion Studio](/docs/studio), you will see a control with the name `rotation` and a slider to change the value.\n\n<img style={{width: '400px'}} src=\"/img/visual-control.png\" />\n\nNow you can change the value as you please, until you have found the optimal value.\n\nOnce you are happy with the value, you can save it back to the code by clicking the save button in the right sidebar.\n\n## Defining a schema\n\nOnly primitive values (`string` and `number`) automatically infer the type of the control.\n\nIf you want to define a more complex schema, you can do so by passing a Zod schema as the third argument.\n\n```tsx twoslash title=\"Editing an object\"\nimport {visualControl} from '@remotion/studio';\n\n// ---cut---\nimport {z} from 'zod';\n\nconst data = visualControl(\n  'my-data',\n  {\n    rotation: 180,\n    text: 'Hello',\n  },\n  z.object({\n    rotation: z.number(),\n    text: z.string(),\n  }),\n);\n```\n\nSee: [Visual Editing](/docs/visual-editing)\n\n```tsx twoslash title=\"Editing a color\"\nimport {visualControl} from '@remotion/studio';\nimport {zColor} from '@remotion/zod-types';\n\nconst color = visualControl('my-color', '#fff', zColor());\n```\n\nSee: [`zColor()`](/docs/zod-types/z-color)\n\n```tsx twoslash title=\"Editing a matrix\"\nimport {visualControl} from '@remotion/studio';\nimport {zMatrix} from '@remotion/zod-types';\n\nconst matrix = visualControl('my-matrix', [1, 2, 3, 4], zMatrix());\n```\n\nSee: [`zMatrix()`](/docs/zod-types/z-matrix)\n\n## When to use\n\n- If you want to create configuration options, use the [Props Editor](/docs/visual-editing).\n- If you want to quickly find the optimal value for a hardcoded constant, use `visualControl()`.\n\n## Important to know\n\n<Step>1</Step> The saving feature works only if the first argument of the `visualControl()` function is static, because static analysis is performed on the source file. The following will not work:\n\n```tsx twoslash title=\"❌ This is not possible\"\nimport {visualControl} from '@remotion/studio';\n\n// ---cut---\n\n// ❌ Not possible to use string interpolation\nconst name = 'my-data';\nconst data = visualControl(`rotation-${name}`, 180);\n\n// ❌ Not possible to use a variable\nconst idOutside = 'rotation-my-data';\nconst dataOutside = visualControl(idOutside, 180);\n```\n\n<Step>2</Step> Controls may not be automatically remove themselves when you remove them from your code.{' '}\n\n## See also\n\n- [Source code for this function](https://github.com/remotion-dev/remotion/blob/main/packages/studio/src/api/visual-control.ts)\n"
