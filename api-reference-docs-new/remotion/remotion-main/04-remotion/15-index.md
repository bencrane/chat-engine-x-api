"---\nimage: /generated/articles-docs-html5-audio.png\ntitle: <Html5Audio>\nid: html5-audio\ncrumb: 'API - remotion'\n---\n\n[_previously called `<Audio>`_](/docs/mediabunny/new-video)\n\nUsing this component, you can add audio to your video. All audio formats which are supported by Chrome are supported by the component.\n\n:::warning[Not supported in client-side rendering]\n`<Html5Audio>` is not supported in [`@remotion/web-renderer`](/docs/client-side-rendering). Use [`<Audio>`](/docs/media/audio) from `@remotion/media` instead.\n:::\n\n## API\n\n### `src`\n\n[Put an audio file into the `public/` folder](/docs/assets) and use [`staticFile()`](/docs/staticfile) to reference it.\n\n```tsx twoslash\nimport {AbsoluteFill, Html5Audio, staticFile} from 'remotion';\n\nexport const MyVideo = () => {\n  return (\n    <AbsoluteFill>\n      <Html5Audio src={staticFile('audio.mp3')} />\n    </AbsoluteFill>\n  );\n};\n```\n\n### `volume?`\n\nAllows you to control the volume of the audio in it's entirety or frame by frame. Read the page on [using audio](/docs/using-audio) to learn more.\n\n```tsx twoslash title=\"Setting a static volume\"\nimport {AbsoluteFill, Html5Audio, staticFile} from 'remotion';\n\nexport const MyVideo = () => {\n  return (\n    <AbsoluteFill>\n      <Html5Audio volume={0.5} src={staticFile('background.mp3')} />\n    </AbsoluteFill>\n  );\n};\n```\n\n```tsx twoslash title=\"Changing the volume over time\"\nimport {AbsoluteFill, Html5Audio, interpolate, staticFile} from 'remotion';\n\nexport const MyVideo = () => {\n  return (\n    <AbsoluteFill>\n      <Html5Audio volume={(f) => interpolate(f, [0, 30], [0, 1], {extrapolateLeft: 'clamp'})} src={staticFile('voice.mp3')} />\n    </AbsoluteFill>\n  );\n};\n```\n\nBy default, volumes between 0 and 1 are supported, where in iOS Safari, the volume is always 1.  \nSee [Volume Limitations](/docs/audio/volume#limitations) for more information.\n\n### `loopVolumeCurveBehavior?`<AvailableFrom v=\"4.0.142\" />\n\nControls the `frame` which is returned when using the [`volume`](#volume) callback function and adding the [`loop`](#loop) attribute.\n\nCan be either `\"repeat\"` (default, start from 0 on each iteration) or `\"extend\"` (keep increasing frames).\n\n### `trimBefore?` / `trimAfter?`<AvailableFrom v=\"4.0.319\"/>\n\n`<Html5Audio>` has two helper props you can use to trim audio:\n\n- `trimBefore` will remove a portion of the audio at the beginning (left side)\n- `trimAfter` will remove a portion of the audio at the end (right side)\n\nIn the following example, we assume that the [`fps`](/docs/composition#fps) of the composition is `30`.\n\nBy passing `trimBefore={60}`, the playback starts immediately, but with the first 2 seconds of the audio trimmed away.  \nBy passing `trimAfter={120}`, any audio after the 4 second mark in the file will be trimmed away.\n\nThe audio will play the range from `00:02:00` to `00:04:00`, meaning the audio will play for 2 seconds.\n\n```tsx twoslash\nimport {AbsoluteFill, Html5Audio, staticFile} from 'remotion';\n\nexport const MyVideo = () => {\n  return (\n    <AbsoluteFill>\n      <Html5Audio src={staticFile('audio.mp3')} trimBefore={60} trimAfter={120} />\n    </AbsoluteFill>\n  );\n};\n```\n\n### ~~`startFrom?` / `endAt?`~~\n\n:::note[Deprecated]\nThese props have been renamed to [`trimBefore`](#trimbefore--trimafter) and [`trimAfter`](#trimbefore--trimafter) in 4.0.319. They will continue to work but you cannot use them together with the new props.\n:::\n\n### `playbackRate?`<AvailableFrom v=\"2.2.0\"/>\n\nYou can use the `playbackRate` prop to control the speed of the audio. `1` is the default and means regular speed, `0.5` slows down the audio so it's twice as long and `2` speeds up the audio so it's twice as fast.\n\nWhile Remotion doesn't limit the range of possible playback speeds, in development mode the [`HTMLMediaElement.playbackRate`](https://developer.mozilla.org/en-US/docs/Web/API/HTMLMediaElement/playbackRate) API is used which throws errors on extreme values. At the time of writing, Google Chrome throws an exception if the playback rate is below `0.0625` or above `16`.\n\n```tsx twoslash\nimport {AbsoluteFill, Html5Audio, staticFile} from 'remotion';\n\nexport const MyVideo = () => {\n  return (\n    <AbsoluteFill>\n      <Html5Audio src={staticFile('audio.mp3')} playbackRate={2} />\n    </AbsoluteFill>\n  );\n};\n```\n\n:::note\nPlaying a video in reverse is not supported.\n:::\n\n### `muted?`<AvailableFrom v=\"2.0.0\"/>\n\nThe `muted` prop will be respected. It will lead to no audio being played while still keeping the audio tag mounted. It's value may change over time, for example to only mute a certain section of the audio.\n\n```tsx twoslash\nimport {AbsoluteFill, Html5Audio, staticFile, useCurrentFrame} from 'remotion';\n\nexport const MyVideo = () => {\n  const frame = useCurrentFrame();\n  return (\n    <AbsoluteFill>\n      <Html5Audio src={staticFile('audio.mp3')} muted={frame < 30} />\n    </AbsoluteFill>\n  );\n};\n```\n\n### `name?`<AvailableFrom v=\"4.0.71\"/>\n\nA name and that will be shown as the label of the sequence in the timeline of the Remotion Studio. This property is purely for helping you keep track of items in the timeline.\n\n### `loop?`<AvailableFrom v=\"3.2.29\"/>\n\nYou can use the `loop` prop to loop audio.\n\n```tsx twoslash\nimport {AbsoluteFill, Html5Audio, staticFile, useCurrentFrame} from 'remotion';\n\nexport const MyVideo = () => {\n  const frame = useCurrentFrame();\n  return (\n    <AbsoluteFill>\n      <Html5Audio loop src={staticFile('audio.mp3')} />\n    </AbsoluteFill>\n  );\n};\n```\n\n### `toneFrequency?`<AvailableFrom v=\"4.0.47\"/>\n\nAdjust the pitch of the audio - will only be applied during rendering.\n\nAccepts a number between `0.01` and `2`, where `1` represents the original pitch. Values less than `1` will decrease the pitch, while values greater than `1` will increase it.\n\nA `toneFrequency` of 0.5 would lower the pitch by half, and a `toneFrequency` of `1.5` would increase the pitch by 50%.\n\n### `audioStreamIndex?`<AvailableFrom v=\"4.0.340\" />\n\nSelect the audio stream to use. The default is `0`.\n\n```tsx twoslash\nimport {AbsoluteFill, Html5Audio} from 'remotion';\n\n// ---cut---\nexport const MyComposition = () => {\n  return (\n    <AbsoluteFill>\n      <Html5Audio audioStreamIndex={1} src={'https://remotion.media/multiple-audio-streams.mov'} />\n    </AbsoluteFill>\n  );\n};\n```\n\n:::note\nThis prop only works during rendering.  \nBrowsers do not support selecting the audio track without enabling experimental flags.\n\nNot to be confused with audio channels. A video can have multiple audio streams, each stream can have multiple channels.  \nMultiple audio streams can be used for example for adding multiple languages to a video.\n\nAudio streams are zero-indexed.\n:::\n\n### `acceptableTimeShiftInSeconds?`<AvailableFrom v=\"3.2.42\"/>\n\nIn the [Remotion Studio](/docs/terminology/studio) or in the [Remotion Player](/docs/player), Remotion will seek the audio if it gets too much out of sync with Remotion's internal time - be it due to the audio loading or the page being too slow to keep up in real-time. By default, a seek is triggered if `0.45` seconds of time shift is encountered. Using this prop, you can customize the threshold.\n\n### `pauseWhenBuffering?`<AvailableFrom v=\"4.0.111\"/>\n\nIf set to `true` and the audio is buffering, the Player will enter into the [native buffering state](/docs/player/buffer-state). The default is `false`, but will become `true` in Remotion 5.0.\n\n### `showInTimeline?`<AvailableFrom v=\"4.0.122\"/>\n\nIf set to `false`, no layer will be shown in the timeline of the Remotion Studio. The default is `true`.\n\n### `delayRenderTimeoutInMilliseconds?`<AvailableFrom v=\"4.0.140\" />\n\nCustomize the [timeout](/docs/delay-render#modifying-the-timeout) of the [`delayRender()`](/docs/delay-render) call that this component makes.\n\n### `delayRenderRetries?`<AvailableFrom v=\"4.0.140\" />\n\nCustomize the [number of retries](/docs/delay-render#retrying) of the [`delayRender()`](/docs/delay-render) call that this component makes.\n\n### `useWebAudioApi?`<AvailableFrom v=\"4.0.306\" />\n\nEnable the [Web Audio API](/docs/audio/volume#limitations) for the audio tag.\n\n### `onError?`<AvailableFrom v=\"4.0.326\" />\n\nHandle an error playing the audio.\n\n### `crossOrigin?`\n\nCorresponds to the [`crossOrigin`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/audio#attr-crossorigin) attribute of the `<audio>` element.\nOne of `\"anonymous\"`, `\"use-credentials\"` or `undefined`.\n\n### ~`allowAmplificationDuringRender?`<AvailableFrom v=\"3.3.17\" />~\n\nDeprecated since v4.0.279: This prop intended to opt into setting the volume to a value higher than one, even though it would only apply during render.\n\nThe option does not make sense anymore, because it is now possible to set the volume higher than `1`.  \nTo prevent synthetic amplification, set a volume not higher than 1.\n\n## Compatibility\n\n<CompatibilityTable chrome firefox safari nodejs=\"\" bun=\"\" serverlessFunctions=\"\" clientSideRendering={false} serverSideRendering player studio
