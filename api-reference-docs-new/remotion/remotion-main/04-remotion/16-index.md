"---\nimage: /generated/articles-docs-html5-video.png\ntitle: <Html5Video>\nid: html5-video\ncrumb: 'API - remotion'\n---\n\n[_previously called `<Video>`_](/docs/mediabunny/new-video)\n\nWraps the native [`<video>`](https://developer.mozilla.org/en-US/docs/Web/API/HTMLVideoElement) element to include video in your component that is synchronized with Remotion's time.\n\n:::note\nPrefer [one of the other video tags](/docs/video-tags) which perform better.\n:::\n\n:::warning[Not supported in client-side rendering]\n`<Html5Video>` is not supported in [`@remotion/web-renderer`](/docs/client-side-rendering). Use [`<Video>`](/docs/media/video) from `@remotion/media` instead.\n:::\n\n## API\n\n[Put a video file into the `public/` folder](/docs/assets) and use [`staticFile()`](/docs/staticfile) to reference it.\n\nAll the props that the native `<video>` element accepts (except `autoplay` and `controls`) will be forwarded (but of course not all are useful for Remotion). This means you can use all CSS to style the video.\n\n```tsx twoslash\nimport {AbsoluteFill, staticFile, Html5Video} from 'remotion';\n\nexport const MyComposition = () => {\n  return (\n    <AbsoluteFill>\n      <Html5Video src={staticFile('video.webm')} />\n    </AbsoluteFill>\n  );\n};\n```\n\nYou can load a video from an URL as well:\n\n```tsx twoslash\nimport {AbsoluteFill, Html5Video} from 'remotion';\n// ---cut---\nexport const MyComposition = () => {\n  return (\n    <AbsoluteFill>\n      <Html5Video src=\"https://remotion.media/BigBuckBunny.mp4\" />\n    </AbsoluteFill>\n  );\n};\n```\n\n## Props\n\n### `src`\n\nThe URL of the video to be rendered. Can be a remote URL or a local file referenced with [`staticFile()`](/docs/staticfile).\n\n### `trimBefore?`<AvailableFrom v=\"4.0.319\"/>\n\nWill remove a portion of the video at the beginning (left side).\n\nIn the following example, we assume that the [`fps`](/docs/composition#fps) of the composition is `30`.\n\nBy passing `trimBefore={60}`, the playback starts immediately, but with the first 2 seconds of the video trimmed away.  \nBy passing `trimAfter={120}`, any video after the 4 second mark in the file will be trimmed away.\n\nThe video will play the range from `00:02:00` to `00:04:00`, meaning the video will play for 2 seconds.\n\nFor exact behavior, see: [Order of operations](/docs/audio/order-of-operations).\n\n```tsx twoslash\nimport {AbsoluteFill, staticFile, Html5Video} from 'remotion';\n\n// ---cut---\nexport const MyComposition = () => {\n  return (\n    <AbsoluteFill>\n      <Html5Video src={staticFile('video.webm')} trimBefore={60} trimAfter={120} />\n    </AbsoluteFill>\n  );\n};\n```\n\n### `trimAfter?`<AvailableFrom v=\"4.0.319\"/>\n\nRemoves a portion of the video at the end (right side). See [`trimBefore`](#trimbefore) for an explanation.\n\n### ~~`startFrom?`~~\n\n:::note[Deprecated]\nThis prop has been renamed to [`trimBefore`](#trimbefore) in 4.0.319. It will continue to work but you cannot use it together with the new prop.\n:::\n\n### ~~`endAt?`~~\n\n:::note[Deprecated]\nThis prop has been renamed to [`trimAfter`](#trimafter) in 4.0.319. It will continue to work but you cannot use it together with the new prop.\n:::\n\n### `style?`\n\nYou can pass any style you can pass to a native `<video>` element. For example, set its size:\n\n```tsx twoslash\nimport {AbsoluteFill, staticFile, Html5Video} from 'remotion';\n\n// ---cut---\nexport const MyComposition = () => {\n  return (\n    <AbsoluteFill>\n      <Html5Video src={staticFile('video.webm')} style={{height: 720, width: 1280}} />\n    </AbsoluteFill>\n  );\n};\n```\n\n### `volume?`\n\nAllows you to control the volume for the whole track or change it on a per-frame basis. Refer to the [using audio](/docs/audio/volume) guide to learn how to use it.\n\n```tsx twoslash title=\"Example using static volume\"\nimport {AbsoluteFill, staticFile, Html5Video} from 'remotion';\n\n// ---cut---\nexport const MyComposition = () => {\n  return (\n    <AbsoluteFill>\n      <Html5Video volume={0.5} src={staticFile('video.webm')} />\n    </AbsoluteFill>\n  );\n};\n```\n\n```tsx twoslash title=\"Example of a ramp up over 100 frames\"\nimport {AbsoluteFill, interpolate, staticFile, Html5Video} from 'remotion';\n\n// ---cut---\nexport const MyComposition = () => {\n  return (\n    <AbsoluteFill>\n      <Html5Video volume={(f) => interpolate(f, [0, 100], [0, 1], {extrapolateLeft: 'clamp', extrapolateRight: 'clamp'})} src={staticFile('video.webm')} />\n    </AbsoluteFill>\n  );\n};\n```\n\nBy default, volumes between 0 and 1 are supported, where in iOS Safari, the volume is always 1.  \nSee [Volume Limitations](/docs/audio/volume#limitations) for more information.\n\n### `loopVolumeCurveBehavior?`<AvailableFrom v=\"4.0.142\" />\n\nControls the `frame` which is returned when using the [`volume`](#volume) callback function and adding the [`loop`](#loop) attribute.\n\nCan be either `\"repeat\"` (default, start from 0 on each iteration) or `\"extend\"` (keep increasing frames).\n\n### `name?`<AvailableFrom v=\"4.0.71\"/>\n\nA name and that will be shown as the label of the sequence in the timeline of the Remotion Studio. This property is purely for helping you keep track of items in the timeline.\n\n### `playbackRate?`<AvailableFrom v=\"2.2.0\" />\n\nControls the speed of the video. `1` is the default and means regular speed, `0.5` slows down the video so it's twice as long and `2` speeds up the video so it's twice as fast.\n\nWhile Remotion doesn't limit the range of possible playback speeds, in development mode the [`HTMLMediaElement.playbackRate`](https://developer.mozilla.org/en-US/docs/Web/API/HTMLMediaElement/playbackRate) API is used which throws errors on extreme values. At the time of writing, Google Chrome throws an exception if the playback rate is below `0.0625` or above `16`.\n\n```tsx twoslash title=\"Example of a video playing twice as fast\"\nimport {AbsoluteFill, staticFile, Html5Video} from 'remotion';\n\n// ---cut---\nexport const MyComposition = () => {\n  return (\n    <AbsoluteFill>\n      <Html5Video playbackRate={2} src={staticFile('video.webm')} />\n    </AbsoluteFill>\n  );\n};\n```\n\n:::note\nPlaying a video in reverse is not supported.\n:::\n\n### `muted?`\n\nYou can drop the audio of the video by adding a `muted` prop:\n\n```tsx twoslash title=\"Example of a muted video\"\nimport {AbsoluteFill, Html5Video} from 'remotion';\n// ---cut---\nexport const MyComposition = () => {\n  return (\n    <AbsoluteFill>\n      <Html5Video muted src=\"https://remotion.media/BigBuckBunny.mp4\" />\n    </AbsoluteFill>\n  );\n};\n```\n\nThis has the benefit that Remotion will not have to download the video file during rendering in order to extract the audio from it.\n\n### `loop?`<AvailableFrom v=\"3.2.29\" />\n\nMakes the video loop indefinitely.\n\n```tsx twoslash title=\"Example of a looped video\"\nimport {AbsoluteFill, Html5Video} from 'remotion';\n// ---cut---\nexport const MyComposition = () => {\n  return (\n    <AbsoluteFill>\n      <Html5Video loop src=\"https://remotion.media/BigBuckBunny.mp4\" />\n    </AbsoluteFill>\n  );\n};\n```\n\n:::note\nWhen a video ends (and `loop` is not set), the last frame of the video remains visible by default.  \nThis is because Remotion mounts a `<video>` tag and sets `ref.currentTime = frame / fps` to synchronize with the timeline. When `currentTime` is set to a value larger than the video's duration, the HTML5 `<video>` element displays the last frame.\n:::\n\n### `acceptableTimeShiftInSeconds?`<AvailableFrom v=\"3.2.42\" />\n\nIn the [Studio](/docs/terminology/studio) or in the [Remotion Player](/docs/player), Remotion will seek the video if it gets too much out of sync with Remotion's internal time - be it due to the video loading or the page being too slow to keep up in real-time. By default, a seek is triggered if `0.45` seconds of time shift is encountered. Using this prop, you can customize the threshold.\n\n### `toneFrequency?`<AvailableFrom v=\"4.0.47\"/>\n\nAdjust the pitch of the audio - will only be applied during rendering.\n\nAccepts a number between `0.01` and `2`, where `1` represents the original pitch. Values less than `1` will decrease the pitch, while values greater than `1` will increase it.\n\nA `toneFrequency` of 0.5 would lower the pitch by half, and a `toneFrequency` of `1.5` would increase the pitch by 50%.\n\n### `audioStreamIndex?`<AvailableFrom v=\"4.0.340\" />\n\nSelect the audio stream to use. The default is `0`.\n\n```tsx twoslash\nimport {AbsoluteFill, Html5Video} from 'remotion';\n\n// ---cut---\nexport const MyComposition = () => {\n  return (\n    <AbsoluteFill>\n      <Html5Video audioStreamIndex={1} src={'https://remotion.media/multiple-audio-streams.mov'} />\n    </AbsoluteFill>\n  );\n};\n```\n\n:::note\nThis prop only works during rendering.  \nBrowsers do not support selecting the audio track without enabling experimental flags.\n\nNot to be confused with audio channels. A video can have multiple audio streams, each stream can have multiple channels.  \nMultiple audio streams can be used for example for adding multiple languages to a video.\n\nAudio streams are zero-indexed.\n:::\n\n### `onError?`\n\nHandle an error playing the video. From v3.3.89, if you pass an `onError` callback, then no exception will be thrown. Previously, the error could not be caught.\n\n### `pauseWhenBuffering?`<AvailableFrom v=\"4.0.100\"/>\n\nIf set to `true` and the video is loading, the Player will enter into the [native buffering state](/docs/player/buffer-state). The default is `false`, but will become `true` in Remotion 5.0.\n\n### `showInTimeline?`<AvailableFrom v=\"4.0.122\"/>\n\nIf set to `false`, no layer will be shown in the timeline of the Remotion Studio. The default is `true`.\n\n### `delayRenderTimeoutInMilliseconds?`<AvailableFrom v=\"4.0.140\" />\n\nCustomize the [timeout](/docs/delay-render#modifying-the-timeout) of the [`delayRender()`](/docs/delay-render) call that this component makes.\n\n### `delayRenderRetries?`<AvailableFrom v=\"4.0.140\" />\n\nCustomize the [number of retries](/docs/delay-render#retrying) of the [`delayRender()`](/docs/delay-render) call that this component makes.\n\n### `onAutoPlayError?`<AvailableFrom v=\"4.0.187\" />\n\nA callback function that gets called when the video fails to play due to autoplay restrictions.  \nIf you don't pass a callback, the video will be muted and be retried once.  \nThis prop is useful if you want to handle the error yourself, e.g. for pausing the Player.  \nRead more here about [autoplay restrictions](/docs/player/autoplay).\n\n### `crossOrigin?`<AvailableFrom v=\"4.0.190\" />\n\nCorresponds to the [`crossOrigin`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/video#attr-crossorigin) attribute of the `<video>` element.  \nOne of `\"anonymous\"`, `\"use-credentials\"` or `undefined`.  \nDefault: `\"anonymous\"` if `onVideoFrame` is specified, `undefined`, otherwise.\n\n### `useWebAudioApi?`<AvailableFrom v=\"4.0.306\" />\n\nEnable the [Web Audio API](/docs/audio/volume#limitations) for the video tag.\n\n### ~`allowAmplificationDuringRender?`<AvailableFrom v=\"3.3.17\" />~\n\nDeprecated since v4.0.279: This prop intended to opt into setting the volume to a value higher than one, even though it would only apply during render.\n\nThe option does not make sense anymore, because it is now possible to set the volume higher than `1`.  \nTo prevent synthetic amplification, set a volume not higher than 1.\n\n## Speed up renders for video with silent audio\n\nRemotion will download the whole video during render in order to mix its audio. If the video contains a silent audio track, you can add the muted property to signal to Remotion that it does not need to download the video and make the render more efficient.\n\n## Codec support\n\nSee: [Which video formats does Remotion support?](/docs/miscellaneous/video-formats)\n\n## Alternatives: `<OffthreadVideo>` and `@remotion/media`\n\n[`<OffthreadVideo>`](/docs/offthreadvideo) is a Rust-based alternative to `<Html5Video>`.  \n[`@remotion/media`](/docs/media/video) is an experimental component that will replace `<Html5Video>` at some point.\n\nTo decide which tag to use, see: [Comparison of video tags](/docs/video-tags)\n\n## Compatibility\n\n<CompatibilityTable chrome firefox safari nodejs=\"\" bun=\"\" serverlessFunctions=\"\" clientSideRendering={false} serverSideRendering player studio hideServers />\n\n## See also\n\n- [Source code for this function](https://github.com/remotion-dev/remotion/blob/main/packages/core/src/video/Video.tsx)\n- [`<Html5Audio />`](/docs/html5-audio)\n- [`<OffthreadVideo />`](/docs/offthreadvideo)\n- [`<Video />` (from `@remotion/media`)](/docs/media/video)\n- [Comparison of video tags](/docs/video-tags)\n- [`Change the speed of a video over time`](/docs/miscellaneous/snippets/accelerated-video)\n"
