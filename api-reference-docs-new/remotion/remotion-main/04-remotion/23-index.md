"---\nimage: /generated/articles-docs-offthreadvideo.png\nid: offthreadvideo\ntitle: '<OffthreadVideo>'\ncrumb: 'API'\n---\n\n<AvailableFrom v=\"3.0.11\" />\n\nThis component imports and displays a video, similar to [`<Html5Video/>`](/docs/html5-video), but during rendering, extracts the exact frame from the video and displays it in a [`<Img>`](/docs/img) tag. This extraction process happens outside the browser using FFmpeg.\n\nThis component was designed to combat limitations of the default `<Html5Video>` element. See: [Comparison of video tags](/docs/video-tags).\n\n:::warning[Not supported in client-side rendering]\n`<OffthreadVideo>` is not supported in [`@remotion/web-renderer`](/docs/client-side-rendering). Use [`<Video>`](/docs/media/video) from `@remotion/media` instead.\n:::\n\n## Example\n\n```tsx twoslash\nimport {AbsoluteFill, OffthreadVideo, staticFile} from 'remotion';\n\nexport const MyVideo = () => {\n  return (\n    <AbsoluteFill>\n      <OffthreadVideo src={staticFile('video.webm')} />\n    </AbsoluteFill>\n  );\n};\n```\n\nYou can load a video from an URL as well:\n\n```tsx twoslash\nimport {AbsoluteFill, OffthreadVideo} from 'remotion';\n// ---cut---\nexport const MyComposition = () => {\n  return (\n    <AbsoluteFill>\n      <OffthreadVideo src=\"https://remotion.media/BigBuckBunny.mp4\" />\n    </AbsoluteFill>\n  );\n};\n```\n\n## Props\n\n### `src`\n\nThe URL of the video to be rendered. Can be a remote URL or a local file referenced with [`staticFile()`](/docs/staticfile).\n\n### `trimBefore?`<AvailableFrom v=\"4.0.319\"/>\n\nWill remove a portion of the video at the beginning (left side).\n\nIn the following example, we assume that the [`fps`](/docs/composition#fps) of the composition is `30`.\n\nBy passing `trimBefore={60}`, the playback starts immediately, but with the first 2 seconds of the video trimmed away.  \nBy passing `trimAfter={120}`, any video after the 4 second mark in the file will be trimmed away.\n\nThe video will play the range from `00:02:00` to `00:04:00`, meaning the video will play for 2 seconds.\n\nFor exact behavior, see: [Order of operations](/docs/audio/order-of-operations).\n\n```tsx twoslash\nimport {AbsoluteFill, OffthreadVideo, staticFile} from 'remotion';\n\n// ---cut---\nexport const MyComposition = () => {\n  return (\n    <AbsoluteFill>\n      <OffthreadVideo src={staticFile('video.webm')} trimBefore={60} trimAfter={120} />\n    </AbsoluteFill>\n  );\n};\n```\n\n### `trimAfter?`<AvailableFrom v=\"4.0.319\"/>\n\nRemoves a portion of the video at the end (right side). See [`trimBefore`](/docs/offthreadvideo#trimbefore) for an explanation.\n\n### ~~`startFrom?`~~\n\n:::note[Deprecated]\nThis prop has been renamed to [`trimBefore`](#trimbefore) in 4.0.319. It will continue to work but you cannot use it together with the new prop.\n:::\n\n### ~~`endAt?`~~\n\n:::note[Deprecated]\nThis prop has been renamed to [`trimAfter`](#trimafter) in 4.0.319. It will continue to work but you cannot use it together with the new prop.\n:::\n\n### `transparent?`<AvailableFrom v=\"4.0.0\" />\n\nIf set to `true`, frames will be extracted as PNG, enabling transparency but also slowing down your render.\n\nIf set to `false` (_default_), frames will be extracted as bitmap (BMP), which is faster.\n\n### `volume?`\n\nAllows you to control the volume for the whole track or change it on a per-frame basis. Refer to the [using audio](/docs/audio/volume) guide to learn how to use it.\n\n```tsx twoslash title=\"Example using static volume\"\nimport {AbsoluteFill, OffthreadVideo, staticFile} from 'remotion';\n\n// ---cut---\nexport const MyComposition = () => {\n  return (\n    <AbsoluteFill>\n      <OffthreadVideo volume={0.5} src={staticFile('video.webm')} />\n    </AbsoluteFill>\n  );\n};\n```\n\n```tsx twoslash title=\"Example of a ramp up over 100 frames\"\nimport {AbsoluteFill, interpolate, staticFile, OffthreadVideo} from 'remotion';\n\n// ---cut---\nexport const MyComposition = () => {\n  return (\n    <AbsoluteFill>\n      <OffthreadVideo volume={(f) => interpolate(f, [0, 100], [0, 1], {extrapolateLeft: 'clamp', extrapolateRight: 'clamp'})} src={staticFile('video.webm')} />\n    </AbsoluteFill>\n  );\n};\n```\n\nBy default, volumes between 0 and 1 are supported, where in iOS Safari, the volume is always 1.  \nSee [Volume Limitations](/docs/audio/volume#limitations) for more information.\n\n### `loopVolumeCurveBehavior?`<AvailableFrom v=\"4.0.142\" />\n\nControls the `frame` which is returned when using the [`volume`](#volume) callback function and wrapping `OffthreadVideo` in a [`<Loop>`](/docs/loop).\n\nCan be either `\"repeat\"` (default, start from 0 on each iteration) or `\"extend\"` (keep increasing frames).\n\n### `style?`\n\nYou can pass any style you can pass to a native HTML element. Keep in mind that during rendering, `<OffthreadVideo>` renders an [`<Img>`](/docs/img) tag, but a `<video>` tag is used during preview.\n\n```tsx twoslash\nimport {AbsoluteFill, Img, staticFile} from 'remotion';\n\n// ---cut---\nexport const MyComposition = () => {\n  return (\n    <AbsoluteFill>\n      <Img src={staticFile('video.webm')} style={{height: 720, width: 1280}} />\n    </AbsoluteFill>\n  );\n};\n```\n\n### `name?`<AvailableFrom v=\"4.0.71\"/>\n\nA name and that will be shown as the label of the sequence in the timeline of the Remotion Studio. This property is purely for helping you keep track of items in the timeline.\n\n### `toneFrequency?`<AvailableFrom v=\"4.0.47\"/>\n\nAdjust the pitch of the audio - will only be applied during rendering.\n\nAccepts a number between `0.01` and `2`, where `1` represents the original pitch. Values less than `1` will decrease the pitch, while values greater than `1` will increase it.\n\nA `toneFrequency` of 0.5 would lower the pitch by half, and a `toneFrequency` of `1.5` would increase the pitch by 50%.\n\n### `onError?`\n\nHandle an error playing the video. From v3.3.89, if you pass an `onError` callback, then no exception will be thrown. Previously, the error could not be caught.\n\n### `playbackRate?`<AvailableFrom v=\"2.2.0\" />\n\nControls the speed of the video. `1` is the default and means regular speed, `0.5` slows down the video so it's twice as long and `2` speeds up the video so it's twice as fast.\n\nWhile Remotion doesn't limit the range of possible playback speeds, in development mode the [`HTMLMediaElement.playbackRate`](https://developer.mozilla.org/en-US/docs/Web/API/HTMLMediaElement/playbackRate) API is used which throws errors on extreme values. At the time of writing, Google Chrome throws an exception if the playback rate is below `0.0625` or above `16`.\n\n```tsx twoslash title=\"Example of a video playing twice as fast\"\nimport {AbsoluteFill, OffthreadVideo, staticFile} from 'remotion';\n\n// ---cut---\nexport const MyComposition = () => {\n  return (\n    <AbsoluteFill>\n      <OffthreadVideo playbackRate={2} src={staticFile('video.webm')} />\n    </AbsoluteFill>\n  );\n};\n```\n\n:::note\nPlaying a video in reverse is not supported.\n:::\n\n### `muted?`\n\nYou can drop the audio of the video by adding a `muted` prop:\n\n```tsx twoslash title=\"Example of a muted video\"\nimport {AbsoluteFill, OffthreadVideo} from 'remotion';\n// ---cut---\nexport const MyComposition = () => {\n  return (\n    <AbsoluteFill>\n      <OffthreadVideo muted src=\"https://remotion.media/BigBuckBunny.mp4\" />\n    </AbsoluteFill>\n  );\n};\n```\n\n### `acceptableTimeShiftInSeconds?`<AvailableFrom v=\"3.2.42\" />\n\nIn the [Studio](/docs/terminology/studio) or in the [Remotion Player](/docs/player), Remotion will seek the video if it gets too much out of sync with Remotion's internal time - be it due to the video loading or the page being too slow to keep up in real-time. By default, a seek is triggered if `0.45` seconds of time shift is encountered. Using this prop, you can customize the threshold.\n\n### `toneFrequency?`<AvailableFrom v=\"4.0.47\"/>\n\nAdjust the pitch of the audio - will only be applied during rendering.\n\nAccepts a number between `0.01` and `2`, where `1` represents the original pitch. Values less than `1` will decrease the pitch, while values greater than `1` will increase it.\n\nA `toneFrequency` of 0.5 would lower the pitch by half, and a `toneFrequency` of `1.5` would increase the pitch by 50%.\n\n### `pauseWhenBuffering?`<AvailableFrom v=\"4.0.111\"/>\n\nIf set to `true` and the video is loading, the Player will enter into the [native buffering state](/docs/player/buffer-state). The default is `false`, but will become `true` in Remotion 5.0.\n\n### `toneMapped?`<AvailableFrom v=\"4.0.117\" />\n\nSince Remotion 4.0.117, Remotion will adjust the colors of videos in different color spaces (such as HDR) when converting to RGB, to counteract color shifts.  \nSince the browser is painting in sRGB, this is necessary to ensure that the colors are displayed correctly.  \nThis behavior is by default `true` and can be disabled by setting `toneMapped` to `false`.  \nDisabling tone mapping will speed up rendering, but may result in less vibrant colors.\n\nPrior to Remotion 4.0.117, tone mapping was always disabled, and the `toneMapped` prop was not available.\n\n### `audioStreamIndex?`<AvailableFrom v=\"4.0.340\" />\n\nSelect the audio stream to use. The default is `0`.\n\n```tsx twoslash\nimport {AbsoluteFill, OffthreadVideo} from 'remotion';\n\n// ---cut---\nexport const MyComposition = () => {\n  return (\n    <AbsoluteFill>\n      <OffthreadVideo audioStreamIndex={1} src={'https://remotion.media/multiple-audio-streams.mov'} />\n    </AbsoluteFill>\n  );\n};\n```\n\n:::note\nThis prop only works during rendering.  \nBrowsers do not support selecting the audio track without enabling experimental flags.\n\nNot to be confused with audio channels. A video can have multiple audio streams, each stream can have multiple channels.  \nMultiple audio streams can be used for example for adding multiple languages to a video.\n\nAudio streams are zero-indexed.\n:::\n\n### `showInTimeline?`<AvailableFrom v=\"4.0.122\"/>\n\nIf set to `false`, no layer will be shown in the timeline of the Remotion Studio. The default is `true`.\n\n### `delayRenderTimeoutInMilliseconds?`<AvailableFrom v=\"4.0.150\" />\n\nCustomize the [timeout](/docs/delay-render#modifying-the-timeout) of the [`delayRender()`](/docs/delay-render) call that this component makes.\n\n### `delayRenderRetries?`<AvailableFrom v=\"4.0.178\" />\n\nCustomize the [number of retries](/docs/delay-render#retrying) of the [`delayRender()`](/docs/delay-render) call that this component makes.\n\n### `onAutoPlayError?`<AvailableFrom v=\"4.0.187\" />\n\nA callback function that gets called when the video fails to play due to autoplay restrictions.  \nIf you don't pass a callback, the video will be muted and be retried once.  \nThis prop is useful if you want to handle the error yourself, e.g. for pausing the Player.  \nRead more here about [autoplay restrictions](/docs/player/autoplay).\n\n### `onVideoFrame?`<AvailableFrom v=\"4.0.190\" />\n\nA callback function that gets called when a frame is extracted from the video.  \nUseful for [video manipulation](/docs/video-manipulation).  \nThe callback is called with a [`CanvasImageSource`](https://developer.mozilla.org/en-US/docs/Web/API/CanvasImageSource) object.  \nDuring preview, this is a `HTMLVideoElement` object, during rendering, it is an `HTMLImageElement`.\n\n### `crossOrigin?`<AvailableFrom v=\"4.0.190\" />\n\nCorresponds to the [`crossOrigin`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/video#attr-crossorigin) attribute of the `<video>` element.  \nOne of `\"anonymous\"`, `\"use-credentials\"` or `undefined`.  \nDefault: `\"anonymous\"` if `onVideoFrame` is specified, `undefined`, otherwise.\n\n### `useWebAudioApi?`<AvailableFrom v=\"4.0.306\" />\n\nEnable the [Web Audio API](/docs/audio/volume#limitations) for the video tag.\n\n### ~~`imageFormat` <AvailableFrom v=\"3.0.22\" />~~\n\n_removed in v4.0.0_\n\nEither `jpeg` or `png`. Default `jpeg`.  \nWith `png`, transparent videos (VP8, VP9, ProRes) can be displayed, however it is around 40% slower, with VP8 videos being [much slower](/docs/slow-method-to-extract-frame).\n\n### ~`allowAmplificationDuringRender?`<AvailableFrom v=\"3.3.17\" />~\n\nDeprecated since v4.0.279: This prop intended to opt into setting the volume to a value higher than one, even though it would only apply during render.\n\nThe option does not make sense anymore, because it is now possible to set the volume higher than `1`.  \nTo prevent synthetic amplification, set a volume not higher than 1.\n\n### Other props\n\nThe props [`onError`](/docs/img#onerror), `className` and `style` are supported and get passed to the underlying HTML element. Remember that during render, this is a `<img>` element, and during Preview, this is a `<video>` element.\n\n## Performance tips\n\nOnly set `transparent` to `true` if you need transparency. It is slower than non-transparent frame extraction.  \nIf you don't care about color accuracy, you can set `toneMapped` to `false` as well to save time on color conversion.\n\n## Looping a OffthreadVideo\n\nUnlike [`<Html5Video>`](/docs/html5-video), `OffthreadVideo` does not implement the `loop` property.  \nConsider using [another video tag](/docs/video-tags) for looping.\n\n:::note\nWhen a video ends, the last frame of the video remains visible by default.  \nThis matches the behavior of [`<Html5Video>`](/docs/html5-video#loop).\n:::\n\nYou can use the following `<LoopableOffthreadVideo>` component that uses Mediabunny to [determine the duration](/docs/mediabunny/metadata) for looping a video.\n\n```tsx twoslash title=\"src/LoopableOffthreadVideo.tsx\"\n// @filename: get-media-metadata.ts\nimport {Input, ALL_FORMATS, UrlSource} from 'mediabunny';\n\nexport const getMediaMetadata = async (src: string) => {\n  const input = new Input({\n    formats: ALL_FORMATS,\n    source: new UrlSource(src, {\n      getRetryDelay: () => null,\n    }),\n  });\n\n  const durationInSeconds = await input.computeDuration();\n  const videoTrack = await input.getPrimaryVideoTrack();\n  const dimensions = videoTrack\n    ? {\n        width: videoTrack.displayWidth,\n        height: videoTrack.displayHeight,\n      }\n    : null;\n  const packetStats = await videoTrack?.computePacketStats();\n  const fps = packetStats?.averagePacketRate ?? null;\n\n  return {\n    durationInSeconds,\n    dimensions,\n    fps,\n  };\n};\n\n// @filename: index.tsx\n// ---cut---\nimport React, {useEffect, useState} from 'react';\nimport {cancelRender, continueRender, delayRender, Loop, OffthreadVideo, RemotionOffthreadVideoProps, useRemotionEnvironment, useVideoConfig, Html5Video} from 'remotion';\nimport {getMediaMetadata} from './get-media-metadata';\n\nconst LoopedOffthreadVideo: React.FC<RemotionOffthreadVideoProps> = (props) => {\n  const [duration, setDuration] = useState<number | null>(null);\n  const [handle] = useState(() => delayRender());\n  const {fps} = useVideoConfig();\n\n  useEffect(() => {\n    getMediaMetadata(props.src)\n      .then(({durationInSeconds}) => {\n        setDuration(durationInSeconds);\n        continueRender(handle);\n      })\n      .catch((err) => {\n        cancelRender(err);\n      });\n\n    return () => {\n      continueRender(handle);\n    };\n  }, [handle, props.src]);\n\n  if (duration === null) {\n    return null;\n  }\n\n  return (\n    <Loop durationInFrames={Math.floor(duration * fps)}>\n      <OffthreadVideo {...props} />\n    </Loop>\n  );\n};\n\nexport const LoopableOffthreadVideo: React.FC<\n  RemotionOffthreadVideoProps & {\n    loop?: boolean;\n  }\n> = ({loop, ...props}) => {\n  const env = useRemotionEnvironment();\n  if (env.isRendering) {\n    if (loop) {\n      return <LoopedOffthreadVideo {...props} />;\n    }\n\n    return <OffthreadVideo {...props} />;\n  }\n\n  return <Html5Video loop={loop} {...props}></Html5Video>;\n};\n```\n\n## Supported codecs by `<OffthreadVideo>`\n\nThe following codecs can be read by `<OffthreadVideo>`:\n\n- H.264 (\"MP4\")\n- H.265 (\"HEVC\")\n- VP8 and VP9 (\"WebM\")\n- AV1 (from _v4.0.6_)\n- ProRes\n\n## This component was designed to combat limitations of the default `<Html5Video>` element. See: [Comparison of video tags](/docs/video-tags).
