"---\nimage: /generated/articles-docs-client-side-rendering-index.png\ncrumb: 'Overview'\nsidebar_label: Overview\ntitle: Client-side rendering\n---\n\n:::warning\nExperimental feature - expect bugs and breaking changes at any time.  \n[Track progress on GitHub](https://github.com/remotion-dev/remotion/issues/5913) and discuss in the [`#web-renderer`](https://remotion.dev/discord) channel on Discord.\n:::\n\nClient-side rendering is a new additional capability of Remotion that allows you to render videos, images, and audio directly in the browser, without requiring server-side infrastructure.\n\nIt is currently released as an experimental alpha under the [`@remotion/web-renderer`](/docs/web-renderer) package.\n\n## Key differences from server-side rendering\n\nUnlike server-side rendering with [`@remotion/renderer`](/docs/renderer), client-side rendering:\n\n- Runs in the browser - No need to setup Node servers or Remotion Lambda\n- Encodes with WebCodecs using [Mediabunny](/docs/mediabunny) instead of FFmpeg\n- Limited to a subset of HTML elements - see [limitations](/docs/client-side-rendering/limitations)\n- No bundling step - takes components and video config directly\n\n## Browser support\n\nClient-side rendering requires the [WebCodecs API](https://developer.mozilla.org/en-US/docs/Web/API/WebCodecs_API), which limits browser support to:\n\n| Browser | Minimum version |\n| ------- | --------------- |\n| Chrome  | 94+             |\n| Firefox | 130+            |\n| Safari  | 26+             |\n\n## APIs\n\nThe package provides APIs called [`renderStillOnWeb()`](/docs/web-renderer/render-still-on-web) and [`renderMediaOnWeb()`](/docs/web-renderer/render-media-on-web).\n\nHere is an example of how to use it:\n\n```tsx twoslash title=\"Tentative API example\"\nimport {renderMediaOnWeb} from '@remotion/web-renderer';\n\nconst Component: React.FC = () => {\n  return (\n    <svg viewBox=\"0 0 100 100\" width=\"100\" height=\"100\" style={{transform: 'rotate(45deg)'}}>\n      <polygon points=\"50,10 90,90 10,90\" fill=\"orange\" />\n    </svg>\n  );\n};\n\nconst {getBlob} = await renderMediaOnWeb({\n  composition: {\n    component: Component,\n    durationInFrames: 100,\n    fps: 30,\n    width: 100,\n    height: 100,\n    calculateMetadata: null,\n    id: 'my-composition',\n  },\n  inputProps: {},\n});\n```\n\n## Enable in the Studio\n\nTo enable client-side rendering in the Studio, you need to call `Config.setExperimentalClientSideRenderingEnabled(true)` in your `remotion.config.ts` file.\n\n```ts twoslash title=\"remotion.config.ts\"\nimport {Config} from '@remotion/cli/config';\n// ---cut---\nConfig.setExperimentalClientSideRenderingEnabled(true);\n```\n\n## Status\n\nClient-side rendering is currently in heavy iteration.\n\n[View the current status and progress →](https://github.com/remotion-dev/remotion/issues/5913)\n\n## See also\n\n- [How it works](/docs/client-side-rendering/how-it-works)\n- [Limitations](/docs/client-side-rendering/limitations)\n- [Server-side rendering](/docs/ssr)\n"
