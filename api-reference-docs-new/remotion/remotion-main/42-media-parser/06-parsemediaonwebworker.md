"---\nimage: /generated/articles-docs-media-parser-parse-media-on-web-worker.png\nid: parse-media-on-web-worker\ntitle: parseMediaOnWebWorker()\nslug: /media-parser/parse-media-on-web-worker\ncrumb: '@remotion/media-parser'\n---\n\n:::warning\n[We are phasing out Media Parser and are moving to Mediabunny](/blog/mediabunny)!\n:::\n\nThis API works the same as [`parseMedia()`](/docs/media-parser/parse-media), except that it executes the parse on a [Web Worker](https://developer.mozilla.org/en-US/docs/Web/API/Web_Workers_API/Using_web_workers). Web Workers are only available in Browsers and in Bun.\n\nThis is useful if you are processing large files which parsing process would block the UI thread for a longer time.\n\nThis API abstracts the communication between the main thread and the worker with the goal to keep the same API as [`parseMedia()`](/docs/media-parser/parse-media), including capabilities like supporting [`mediaParserController()`](/docs/media-parser/media-parser-controller) all async callback functions.\n\nThe only difference in API is that the [`reader`](/docs/media-parser/parse-media#reader) option is not available. This is because you cannot pass a function to a Web Worker.\n\n```tsx twoslash title=\"Parsing a video on a Web Worker\"\nimport {parseMediaOnWebWorker} from '@remotion/media-parser/worker';\n\nconst result = await parseMediaOnWebWorker({\n  src: 'https://remotion.media/video.mp4',\n  fields: {\n    durationInSeconds: true,\n    dimensions: true,\n  },\n});\n\nconsole.log(result.durationInSeconds); // 10\nconsole.log(result.dimensions); // {width: 1920, height: 1080}\n```\n\n## API\n\nSame as [`parseMedia()`](/docs/media-parser/parse-media), but without the option to pass a [`reader`](/docs/media-parser/parse-media#reader).\n\nThe [`reader`](/docs/media-parser/parse-media#reader) option in the worker is hardcoded to [`webReader`](/docs/media-parser/web-reader).\n\n## See also\n\n- [Source code for this function](https://github.com/remotion-dev/remotion/blob/main/packages/media-parser/src/worker.module.ts)\n- [`parseMedia()`](/docs/media-parser/parse-media)\n"
