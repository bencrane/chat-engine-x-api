'@remotion/media-parser'\n---\n\n:::warning\n[We are phasing out Media Parser and are moving to Mediabunny](/blog/mediabunny)!\n:::\n\nSame as [`parseMedia()`](/docs/media-parser/parse-media), but also downloads the media file to disk.\n\nMeant to be used in Node.js and Bun.\n\n```tsx twoslash title=\"Download a file\"\nimport {downloadAndParseMedia} from '@remotion/media-parser';\nimport {nodeWriter} from '@remotion/media-parser/node-writer';\n\nawait downloadAndParseMedia({\n  src: 'https://www.w3schools.com/html/mov_bbb.mp4',\n  writer: nodeWriter('output.mp4'),\n});\n```\n\nYou can obtain [fields](/docs/media-parser/parse-media#fields) like `tracks` and `duration` by passing them to the `fields` object.\n\n```tsx twoslash title=\"Download a file and get metadata\"\nimport {downloadAndParseMedia} from '@remotion/media-parser';\nimport {nodeWriter} from '@remotion/media-parser/node-writer';\n\nconst {durationInSeconds, tracks} = await downloadAndParseMedia({\n  src: 'https://s3.amazonaws.com/bucket/uploaded-asset.mp4',\n  writer: nodeWriter('output.mp4'),\n  fields: {\n    durationInSeconds: true,\n    tracks: true,\n  },\n});\n// If here was reached, file is downloaded!\nconsole.log(durationInSeconds);\nconsole.log(tracks);\n```\n\nYou can use callback functions to retrieve information as soon as it is available.  \nThrow an error to stop the download.\n\n```tsx twoslash title=\"Stop the download if the video is too long\"\nimport {downloadAndParseMedia} from '@remotion/media-parser';\nimport {nodeWriter} from '@remotion/media-parser/node-writer';\n\nawait downloadAndParseMedia({\n  src: 'https://s3.amazonaws.com/bucket/uploaded-asset.mp4',\n  writer: nodeWriter('output.mp4'),\n  onDurationInSeconds: (duration) => {\n    if (duration && duration > 600) {\n      throw new Error('Video is too long');\n    }\n  },\n});\n```\n\nIf an error occurs (including one you've thrown yourself), you can decide what to do using [`onError`](#onerror).\n\n```tsx twoslash title=\"Continue download despite error\"\nimport {downloadAndParseMedia} from '@remotion/media-parser';\nimport {nodeWriter} from '@remotion/media-parser/node-writer';\n\nawait downloadAndParseMedia({\n  src: 'https://s3.amazonaws.com/bucket/uploaded-asset.mp4',\n  writer: nodeWriter('output.mp4'),\n  onError: (error) => {\n    // Force the file to be downloaded despite parsing error.\n    // Note: At the end, the error will be thrown nonetheless.\n    return {action: 'download'};\n\n    // Default behavior:\n    // Abort the download, delete the file and throw the error immediately.\n    // return {action: 'fail'};\n  },\n});\n```\n\n## API\n\nAll of the same parameters for [`parseMedia()`](/docs/media-parser/parse-media) are available, plus:\n\n### `writer`\n\nThe writer to use to write the downloaded file to disk. Currently available:\n\n- `nodeWriter` from `@remotion/media-parser/node-writer`: Writes to disk using Node.js's `fs` module.\n\n### `onError`\n\nA function that is called when an error occurs. It receives the error as an argument.  \nYou must return one of the following:\n\n- `{action: 'download'}`: Continue downloading the file despite the error.\n- `{action: 'fail'}`: Abort the download, delete the file and throw the error immediately.\n\nSee also the example above.  \nThe function may be async, parsing is paused until it resolves.\n\n## See also\n\n- [Source code for this function](https://github.com/remotion-dev/remotion/blob/main/packages/media-parser/src/download-and-parse-media.ts)\n- [`parseMedia()`](/docs/media-parser/parse-media)\n"
