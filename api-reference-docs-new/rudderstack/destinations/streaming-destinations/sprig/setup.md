# Setup Guide

Set up Sprig as a destination in RudderStack.

* * *

  * __2 minute read

  * 


This guide will help you set up Sprig as a destination in the RudderStack dashboard. It also lists the configuration settings required to correctly send data from the supported sources to Sprig.

## Setup

  1. In your [RudderStack dashboard](<https://app.rudderstack.com/>), go to **Directory** > **Destinations** > **Cloud Destinations** and search for **Sprig**.
  2. Connect your source and click **Continue**.


### Connection settings

Configure the following settings to set up Sprig as a destination in RudderStack:

  * **Name** : Assign a name to uniquely identify the destination.
  * **Public API Key** : Enter your Sprig API key by logging in to your [Sprig dashboard](<https://app.sprig.com/login>) and going to **Integrations** > **Enrichment** > **RudderStack**.
  * **Environment ID** : Enter your Sprig environment ID. You can find it in your Sprig dashboard by going to **Integrations** > **Installation** > **JavaScript**. Note that this setting is applicable only when sending events in [web device mode](<https://www.rudderstack.com/docs/destinations/streaming-destinations/sprig/device-mode/>).

### Configuration settings

  * **Client-side Events Filtering** : This setting lets you specify which events should be blocked or allowed to flow through to Sprig. For more information on this setting, see [Client-side Events Filtering](<https://www.rudderstack.com/docs/sources/event-streams/sdks/event-filtering/>).
  * **Consent management settings** : Configure the consent management settings for the specified source by choosing the **Consent management provider** from the dropdown and entering the relevant consent category IDs. See [Consent Management in RudderStack](<https://www.rudderstack.com/docs/data-governance/consent-management/overview/>) for more information on this feature.
