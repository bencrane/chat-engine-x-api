# How to Report Issues Effectively

## Overview

The documentation emphasizes that "specific and actionable details when reporting issues" matter significantly, as vague messages like "X is not working" create delays in resolution.

## Three Issue Categories

The guide covers three main types:

1. **Assistant-Related Issues** – Voice problems, transcription errors, call failures
2. **Dashboard & UI Issues** – Interface and functionality problems
3. **Account-Related Issues** – Login, billing, verification concerns

## Assistant-Related Reporting

When documenting assistant problems, include:
- Call ID from your test session
- Precise timestamp of the problem
- Clear description with expected versus actual behavior
- Screen recording for web-based calls

Call IDs can be retrieved through the dashboard at https://dashboard.vapi.ai/calls. The guide notes that "if the call never connected or failed completely, simply share the Call ID" for investigation.

## Dashboard & UI Reporting

For interface issues, provide:
- Screen recordings or screenshots
- Browser and operating system details
- Reproduction steps
- Console error messages (accessible via F12)

The documentation stresses recording "the complete workflow" with audio narration explaining the problem.

## Account-Related Reporting

Include your account email, Organization ID (found at https://dashboard.vapi.ai/org/settings), and relevant error screenshots.

## Key Recommendation

The guide advises users to "be specific" rather than vague, maintain documentation of modifications, and recognize that "screen recordings are invaluable for UI and workflow issues."
